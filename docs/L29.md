 第29课：Attribute Table 是什么，为什么背景颜色不是按每个 tile 单独存的 

 这节课要解决一个非常自然的问题： 

 你已经知道： 

  - Name Table 决定“这里放哪个 tile” 
  - Pattern Table 决定“tile 长什么样” 
    

 那颜色呢？ 

 你可能会本能以为： 

  - 每个 tile 自己再存一份颜色信息 
    

 但 NES 不是这么干的。 

 先给核心结论： 

  - 背景颜色不是按每个 tile 单独存 
  - 而是按更大的区域分组存放 
  - 这个分组信息就放在 Attribute Table 
    

 所以 Attribute Table 的作用是： 

  - 告诉 PPU：这片背景区域应该使用哪组背景调色板 
    
---------

1. 先回顾背景的三层信息 

 背景最终能显示出来，至少需要三类信息： 
  1. 这个位置放哪个 tile 
     来自 Name Table 
  2. 这个 tile 的图形位是什么 
     来自 Pattern Table 
  3. 这个 tile 显示时该用哪组颜色 
     来自 Attribute Table + Palette 
     

 所以 Attribute Table 本质上是在补齐“颜色选择”这一层。 

---------

2. 为什么不按每个 tile 单独存颜色 
   

 因为 NES 很节省空间。 

 如果每个 tile 都再配一份独立颜色信息，开销就会更大。 
 于是 NES 采用了一个折中设计： 

  - 颜色不是细到每个 tile 
  - 而是按一个更大的背景区域共享 
    

 这样能节省存储空间，但代价是颜色控制粒度更粗。 

 这是一种非常典型的老硬件设计思路： 

  - 用更粗粒度的控制换取更省资源 
    
---------

3. Attribute Table 在 Name Table 的哪里 
   

 每张 Name Table 是 1KB： 

  - 前面大部分区域是 tile 编号 
  - 最后有一小块区域是 Attribute Table 
    

 也就是说： 

  - Name Table 不只是“tile 地图” 
  - 它尾部还附带了这张背景地图对应的颜色区域信息 
    

 所以你可以把一张 Name Table 粗略理解为： 

  - 大部分：图块编号 
  - 最后一小块：颜色分区描述 
    
---------

4. Attribute Table 控制的不是 1 个 tile，而是一片区域 
   

 这是这节课最关键的地方。 

 Attribute Table 的一个条目，不是对应一个 tile。 
 它对应的是更大的背景块。 

 你当前阶段先记这个结论就够： 

  - Attribute Table 的颜色控制粒度比 tile 粗 
  - 所以同一片区域里的多个 tile 会共享某一组调色信息 
    

 这就是为什么 NES 背景颜色会有那种“块状分区感”。 

---------

5. 为什么说它决定的是“用哪组调色板” 

 这里再细一点，但仍保持最小理解。 

 Attribute Table 存的不是最终 RGB 颜色。 
 它存的是更像“编号”： 

  - 选背景调色板 0 
  - 或调色板 1 
  - 或调色板 2 
  - 或调色板 3 

 然后真正具体颜色，还要再去 Palette 区查。 

 所以 Attribute Table 更像： 

  - 颜色组选择器 
  - 而不是最终颜色值本身 
    
---------

6. 背景像素最终是怎么被决定的 

 你现在可以把背景像素的来源想成三步： 
  1. 从 Name Table 找到 tile 编号 
  2. 从 Pattern Table 找到该 tile 某个像素的图形位 
  3. 从 Attribute Table 找到这块区域该用哪组背景调色板 
     

 然后这三者组合起来，最终才得到： 
  - 这个屏幕像素应该显示什么颜色 

 所以背景渲染不是“查一张表”，而是多张表组合。 

---------

7. 为什么这会让 PPU 比普通 framebuffer 麻烦很多 

 如是普通 framebuffer： 
  - 每个像素直接存颜色 

 但 NES 背景不是这样。 

 它更像： 

  - 先查地图 
  - 再查图块像素 
  - 再查颜色区域 
  - 再查调色板 

 所以你现在看到的 Name Table、Pattern Table、Attribute Table、Palette，实际上是在联合构造最终像素。 

 这也是为什么 PPU 天生更像“图像生成器”，而不是“像素数组”。 

---------

8. 当前阶段你最该建立的直觉 

 你可以先把 Attribute Table 想成： 
  - 背景地图上的“颜色分区图” 

 Name Table 说： 
  - 这里摆什么砖块 

 Attribute Table 说： 
  - 这片砖块区域用哪种配色 

 这个类比非常适合现在。 

---------

9. 这节课和 mirroring 的关系 

 你前一课刚学了 Name Table mirroring。 
 现在你要意识到： 

  - 一张 Name Table 不只是 tile 编号区域 
  - 还附带 Attribute Table 
    

 所以当 Name Table 被镜像时，Attribute Table 也跟着这张背景表一起被镜像。 

 这就是为什么 mirroring 不仅影响“地图内容”，也会影响“颜色区域布局”。 

---------

10. 当前阶段你还不需要死记的细节 

 你现在先不用强行背： 
  - 一个 Attribute 字节具体控制几乘几 tile 
  - 每两位如何切成四个小块 
  - 每块如何对应 palette 选择位 

 这些是下一阶段更细的 PPU 背景取数细节。 
 现在你先把“大图景”稳住： 

  - Attribute Table 是背景颜色分区信息 
  - 它的粒度比 tile 粗 
  - 它选的是调色板组，不是最终 RGB 
    
---------

11. 这节课的最简总结 

  - Name Table 决定放什么 tile 
  - Pattern Table 决定 tile 长什么样 
  - Attribute Table 决定这片区域用哪组背景调色板 
  - Palette 决定最终颜色值 
  - 所以背景颜色不是“每 tile 独立存一份”，而是按区域共享 
    
---------
