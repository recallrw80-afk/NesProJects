第30课：Sprite 是怎么组成的，OAM 那 256 字节到底在描述什么

 这节课终于把你前面写过的 OAM DMA、OAMADDR、OAMDATA 真正讲明白。 

 先给核心结论： 

  - OAM 里存的不是像素 
  - OAM 存的是精灵描述信息 
  - PPU 会根据这些描述信息，再去 Pattern Table 取图，再结合属性把精灵画出来 

 所以 OAM 更像： 

  - “精灵清单” 
  - 而不是“精灵图像数据本体” 
     
---------

    1. Sprite 是什么 

 Sprite 可以先理解成： 
  - 出现在背景上方或下方的独立小图块对象 
  - 通常用来表示角色、敌人、子弹、道具等 

 它和背景的区别是： 
  - 背景来自 Name Table 地图系统 
  - Sprite 是“单独摆放”的对象系统 

 所以你可以把 NES 画面理解成两层： 
  1. 背景层 
    2. 精灵层 

 当然真实情况还有优先级细节，但当前阶段这样理解就够。 
 
---------

    2. OAM 为什么是 256 字节 

因为标准 NES 一共有 64 个 sprite 条目，每个 sprite 用 4 字节 描述： 

64 * 4 = 256 字节 

这就是为什么 OAM DMA 总是一次搬 256 字节。 

不是因为“刚好方便”，而是因为： 

  - 整张精灵表刚好就是 256 字节 
     
---------

  3. 每个 Sprite 的 4 字节分别是什么 
      

 这是这节课最重要的内容，你直接记住： 

 第 1 字节：Y 坐标 
 第 2 字节：tile 编号 
 第 3 字节：属性 
 第 4 字节：X 坐标 

 也就是说，一个 sprite 条目长这样： 

 [Y, tile_index, attributes, X] 

 你必须把这个顺序背下来。 

---------

  4. 这 4 字节分别是什么意思 
      
  ### 1. Y 坐标 

 表示精灵垂直位置。 
 
  ### 2. tile 编号 

 表示这个精灵要用 Pattern Table 里的哪一个 tile 图形。 
 
  ### 3. 属性 

 这是一个信息打包字段，里面通常包含： 
 
  - 调色板选择 
  - 是否水平翻转 
  - 是否垂直翻转 
  - 优先级等 
     

 你现在先记住： 
 
  - 属性字节不是颜色本身 
  - 它是“精灵显示规则”的集合 
     
  ### 4. X 坐标 

 表示精灵水平位置。 
 
---------

    5. 为什么 OAM 不存像素 

 因为像素图已经在 Pattern Table 里了。 

 OAM 的职责不是“重复存图”，而是告诉 PPU： 

  - 把哪张 tile 图 
  - 放到哪里 
  - 用什么属性 
  - 以什么方式显示 

 所以 OAM 本质上就是“引用 + 位置 + 显示规则”。 

 这和你前面学的背景系统是类似的： 

  - 背景不是直接存像素，而是 tile 编号 + 图块数据 + 调色板 
  - 精灵也不是直接存像素，而是 tile 编号 + 坐标 + 属性 

 NES 很少直接存最终像素，它更喜欢“查表组合”。 
 
---------

    6. 你现在终于能理解 OAM DMA 为什么这么重要了 

因为每一帧游戏都可能要更新很多精灵： 
  - 人物走路 
  - 子弹飞行 
  - 敌人移动 
  - UI 小图标变化 

如果 CPU 自己一字节一字节慢慢写 OAM，会很麻烦。 
 于是有了 OAM DMA： 

  - 一次把 256 字节整张精灵表搬过去 

这等于每帧快速刷新“本帧所有 sprite 的描述信息”。 

所以 OAM DMA 不是偶然设计，它和 sprite 系统天然配套。 
 

---------

    7. 你可以这样理解背景和精灵的区别 

 背景 
  - 来自 Name Table 
  - 更像地图 
  - 连续铺满画面 

 精灵 
  - 来自 OAM 
  - 更像对象列表 
  - 单独摆放在屏幕各处 

 这两个系统共同组成最终画面。 
 
---------

    8. 属性字节为什么很重要 

 虽然你现在还不用背每一位，但你必须知道： 
  - Sprite 的“显示样式”很多不由 tile 图本身决定 
  - 而由属性字节决定

 也就是说，同一个 tile 编号： 
  - 可以翻转 
  - 可以换 palette 
  - 可以改变前后层级关系 

 所以属性字节让精灵显示更灵活。

---------

  10. 当前阶段你最应该记住的图 
       

 一个 sprite = 4 字节： 

 byte0: Y 
 byte1: tile index 
 byte2: attributes 
 byte3: X 

 一共 64 个 sprite： 

 64 * 4 = 256 bytes 

 这就是 OAM 的本质。 

---------

  11. 这节课的最简总结 
       
  - OAM 是精灵描述表，不是像素表 
  - 每个 sprite 用 4 字节描述
  - 内容是：Y, tile, attributes, X 
  - PPU 根据 OAM 去取 Pattern Table 图形并显示精灵 
  - OAM DMA 一次搬 256 字节，是因为整张 sprite 表刚好就是 256 字节
