第 8 课：CPU 总线与地址空间

 前面你已经知道：

  - CPU 通过地址访问系统
  - Bus 负责解释地址
  - 地址空间不等于一整块 RAM

 这一课我们把它系统化，真正把 CPU 看到的整张“地址地图”建立起来。

 这是后面理解：

  - PPU 寄存器
  - 手柄输入
  - DMA
  - 卡带 ROM
  - Mapper

 的共同基础。

---------

    1. CPU 能看到多大的地址空间 

 NES CPU 是 16 位地址总线，所以 CPU 能发出的地址范围是： 
  - 0x0000..0xFFFF 

 总共 64KB 

 这叫：CPU 地址空间 

 注意，这还是那句老话： 

 地址空间 = 可寻址范围 
 不是 = 真有 64KB 统一 RAM 

 CPU 只是能说出这些门牌号。 
 这些门牌号具体由谁响应，要看映射规则。 

---------

    2. CPU 地址空间的大致分区

 文档第 4.7 节其实已经把这个分区讲出来了，对应实现也能在 core/bus.c 里看到。 

 你现在先记这个版本：

  - 0x0000..0x1FFF：内部 RAM 及其镜像 
  - 0x2000..0x3FFF：PPU 寄存器及其镜像 
  - 0x4000..0x401F：APU 和 I/O 
  - 0x4020..0x5FFF：扩展/未使用/部分 Mapper 区域 
  - 0x6000..0x7FFF：卡带 RAM，常叫 PRG RAM / WRAM 
  - 0x8000..0xFFFF：卡带 PRG ROM 

 这张图你以后要非常熟。 
 因为 CPU 执行的大部分事情，最后都可以归结成： 
 它现在访问的是这张地图的哪一块 

---------

  3. 0x0000..0x1FFF：内部 RAM 和镜像 

 NES 真正的内部 RAM 只有 2KB，也就是： 
  - 0x0000..0x07FF 

 但 CPU 地址空间里给它分配到了： 
  - 0x0000..0x1FFF 

 为什么这么大？ 

 因为后面很多地址其实只是前面 2KB 的重复镜像。 

 这个项目里用的就是： 

 core/bus.c:59 

 __ram[addr & 0x7FF] 

 这意味着： 

  - 0x0000 
  - 0x0800 
  - 0x1000 
  - 0x1800 

 最后都能落到同一块 RAM 上的对应位置。 

 所以你现在看到 0x17A3 这种地址，不要以为一定是“更高的新 RAM”，它很可能只是镜像回去。 

---------

    4. 0x2000..0x3FFF：PPU 寄存器和镜像 

 这是 NES 初学者最容易混的区域之一。 
 PPU 真实暴露给 CPU 的主要寄存器其实只有 8 个，常见是： 

  - 0x2000 
  - 0x2001 
  - 0x2002 
  - 0x2003 
  - 0x2004 
  - 0x2005 
  - 0x2006 
  - 0x2007 

 但地址空间给到了： 
  - 0x2000..0x3FFF 

 所以这中间也存在镜像。 
 也就是说： 

  - 0x2008 可能镜像回 0x2000 
  - 0x2009 可能镜像回 0x2001 

 你现在不用背每个寄存器功能，先只要知道： 
 这整片区域不是普通内存，而是 CPU 与 PPU 通信的窗口 
 所以： 

  - LDA $2002 可能是在读 PPU 状态 
  - STA $2000 可能是在写 PPU 控制寄存器 
---------

    5. 0x4000..0x401F：APU 和 I/O 

 这段区域主要给： 
  - APU 
  - 手柄输入 
  - DMA 
  - 其他 I/O 

 你已经见过两个关键地址了： 
  - 0x4014：OAM DMA 
  - 0x4016/0x4017：手柄寄存器 

 这个项目里也明确单独分出来了：core/bus.c:11 

 所以这段区域的特点是： 

 很多地址不是在“存数据”，而是在和设备对话 

 比如： 

  - 写 0x4014 是启动 DMA 
  - 读 0x4016 是读手柄移位寄存器 
  - 写某些 0x400x 地址是在控制 APU 发声参数 

6. 0x4020..0x5FFF：扩展/未使用区域 

 这段你现在先知道有这么块地方就行。 
 它可能涉及： 

  - 扩展硬件 
  - 一些 Mapper 扩展寄存器 
  - 某些卡带特殊行为 

 现在不用深挖。 
 但你要知道 NES 地址空间里并不是每一段都同样“常用”。 

---------

7. 0x6000..0x7FFF：卡带 RAM 

 这段通常给卡带上的 RAM，用于： 
  - 存档 
  - 工作数据
  - 某些游戏运行时缓冲 

 有些 ROM 会带电池，这样断电后还能保存内容。 
 所以这段不是主机内部 RAM，而是： 
 卡带自己带来的 RAM 
 这点很重要，因为以后你会发现： 

  - 不是所有 RAM 都是 NES 主机本体的 
  - 卡带也可能带自己的存储区域 

8. 0x8000..0xFFFF：PRG ROM 

 这是 CPU 最关心的高地址区域。 
 大多数情况下，CPU 的程序执行会大量落在这里。 
 因为这里通常映射到卡带的 PRG ROM。 
 也就是说： 

  - CPU 从这里取 opcode 
  - 从这里取常量数据 
  - 从这里读程序表 

 所以你以后看到 reset vector、程序入口、主循环代码，很多都和这片区域有关。 
 但有个重点： 
 这片区域虽然通常是 PRG ROM，但具体映射哪一页，可能由 Mapper 决定 
 我们暂时不展开 Mapper，只先记住这是卡带程序区。 

---------

9. 为什么说地址空间里混着“内存”和“设备” 

 这是模拟器的一个核心难点。 
 在现代应用开发里，你通常觉得： 

  - 内存是内存 
  - 设备是设备 

 但 NES 这种机器里，CPU 眼里都只是“地址”。 
 于是地址空间里会混着： 

  - 真实 RAM 
  - 只读 ROM 
  - PPU 寄存器 
  - APU 寄存器 
  - 手柄寄存器 
  - DMA 寄存器 

 这就意味着： 

 CPU 执行 read/write 时，不只是读写存储，也可能是在操作硬件状态 

 这也是为什么总线不能只是一个 Vec\<u8>。 

---------

10. 总线在软件里到底承担什么职责 

 你前面已经回答过一部分，现在系统化一下。 

 总线在模拟器里通常至少做这几件事： 

  1. 根据地址判断目标设备 
  2. 处理镜像 
  3. 把读写转发给对应模块 
  4. 维持某些共享状态 
  5. 作为 CPU 访问整个系统的统一入口 

 所以从软件结构上说： 
 Bus 是 CPU 和外部世界之间的适配层 
 它让 CPU 只关心： 
 read(addr) 
 write(addr, value) 
 而不用关心： 

  - 这个地址是不是 PPU 
  - 是不是镜像 
  - 是不是卡带 
  - 会不会触发 DMA 
  - 会不会读到手柄移位寄存器 
    
---------

11. 用两个例子把地址空间想透 

 例子 1：CPU 读 0x0002 

 流程： 

  1. CPU 发出地址 0x0002 
  2. Bus 判断：这是 RAM 区 
  3. 从内部 2KB RAM 里读偏移 0x0002 
  4. 返回给 CPU 

 这是比较普通的情况。 
 例子 2：CPU 写 0x4014 
 流程： 

  1. CPU 发出写地址 0x4014 
  2. Bus 判断：这是 DMA 寄存器 
  3. 不是普通内存写 
        4. 启动 DMA 逻辑 

 这个例子很能说明： 
 CPU 看到的都是地址，但地址背后的语义完全可能不同 

---------

12. Rust 里你将来怎么表示这张地图

 你后面很可能会写出类似这种接口： 

```rust
impl Bus { 
    pub fn read(&mut self, addr: u16) -> u8 { 
        match addr { 
            0x0000..=0x1FFF => { ... } 
            0x2000..=0x3FFF => { ... } 
            0x4014 => { ... } 
            0x4016..=0x4017 => { ... } 
            0x6000..=0x7FFF => { ... } 
            0x8000..=0xFFFF => { ... } 
            _ => 0, 
        } 
    } 
}  
```

 这里最重要的不是语法，而是： 

 你在代码里重建 CPU 的地址地图 

 所以实现总线，本质上是在把 NES 的内存地图翻译成 Rust 逻辑。


---------

  13. 这节课最重要的认知 
      

 你现在一定要彻底接受一件事： 

 NES 的地址空间不是“很多内存块排在一起”，而是一张由硬件规则决定的功能地图 

 这张地图上： 

  - 有些格子是 RAM 
  - 有些格子是 ROM 
  - 有些格子是 PPU 窗口 
  - 有些格子是输入设备 
  - 有些格子是 DMA 开关 
  - 有些格子是卡带自己扩展出来的区域 
    

 这就是模拟器和普通应用最不一样的地方之一。 

---------

  14. 这节课你必须记住的 6 句话 
      

- CPU 地址空间是 0x0000..0xFFFF，共 64KB。 

- 地址空间不等于统一连续 RAM。 

- 0x0000..0x1FFF 是内部 RAM 及镜像。 

- 0x2000..0x3FFF 是 PPU 寄存器窗口及镜像。 

- 0x4000..0x401F 里混着 APU、输入和 DMA 等 I/O。 

- 0x8000..0xFFFF 通常是卡带 PRG 区。
