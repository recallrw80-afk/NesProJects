第 6 课：寻址模式 

 这节课非常关键。 
 很多人一看到 6502 指令就去背 LDA/STA/ADC/...，结果越学越乱。真正更重要的，是先理解： 

 这条指令操作的数据，到底从哪里来？ 

 这就是寻址模式。 

 你可以把一条指令拆成两部分： 

  - 做什么：比如加载、存储、加法、跳转
  - 对谁做：数据在哪里 
    

 前者是 opcode 的“动作”，后者就是寻址模式。 

 所以： 

 *LDA #$10* 和 *LDA $2000* 动作都是 LDA

 这节课只学“对谁做”。 

---------

1. 什么是寻址模式 


 寻址模式就是：

 CPU 如何找到本次指令要操作的数据或地址 

 比如同样是 LDA： 

  - LDA #\$10 
    直接把 0x10 当数据 
  - LDA \$20 
    先把 \$20 当地址，再去这个地址读数据 
  - LDA \$2000,X 
    先算地址 \$2000 + X，再去读数据 

 所以你以后看一条指令时，要养成先问这个问题： 

 这条指令的操作数是值本身，还是一个地址，还是一个地址的地址？

---------

2. 为什么寻址模式这么重要 


 因为 CPU 执行一条指令，很多时候真正复杂的不是“做什么”，而是“去哪找数据”。 

 比如： 

  - LDA 的核心动作很简单，就是 A = value 
  - 难点在于 value 从哪来 
    

 所以后面写 CPU 时，你会发现很自然的一种拆分方式是： 

  1. 先根据寻址模式算出操作数位置 
  2. 再执行指令逻辑 


 这也是很多模拟器实现 CPU 的常见结构。 

---------

3. 先建立一个总分类 


 6502 常见寻址模式，你现在先粗分成 4 类： 

1. 没有显式操作数 

  - 隐含寻址 IMP 
    
    2. 操作数就是值 
    
  - 立即数 IMM
    
    3. 操作数里给的是地址 
    
  - 零页 ZP0 
  - 零页,X ZPX 
  - 零页,Y ZPY 
  - 绝对 ABS 
  - 绝对,X ABX 
  - 绝对,Y ABY 
    
    4. 操作数还要再转一次 
    
  - 相对 REL 
  - 间接 IND 
  - (zp,X) 即 IZX 
  - (zp),Y 即 IZY 
    

 你现在先不要怕名字多，后面一个个拆。 

---------

4. IMP：隐含寻址 


 IMP = Implied 

 意思是： 

 这条指令不需要额外操作数，操作对象已经隐含在指令定义里 

 比如： 

  - INX 
    默认就是 X = X + 1 
  - CLC 
    默认就是清除 C 标志 

 这类指令不需要“再从 ROM 后面多读一个字节当参数”。 

 所以你可以理解成： 

 指令自己已经把目标写死了 

---------

5. IMM：立即数 


 IMM = Immediate 

 最容易理解。 

 比如： 

 LDA #\$10 

 这里的 #\$10 就是字面值本身。 
 CPU 不需要再把 0x10 当地址去访问内存，而是直接把它当数据。 

所以： 

- \# 非常重要 
- 有 \# 通常表示“后面的就是值本身” 


执行效果就是：

a = 0x10; 

不是“去地址 0x0010 读”。 

这是初学者最容易混的点。 

---------

6. ZP0：零页寻址 


 ZP0 = Zero Page 

 比如：

 LDA $20 

 这里 $20 不是值，而是地址。 
 而且它是一个 8 位地址，所以范围在： 

  - 0x00..0xFF 
    

 也就是“第 0 页”内存，所以叫零页。 

 零页为什么特别？ 

 因为： 

  - 地址只要 1 个字节 
  - 访问快 
  - 早期程序很喜欢用它当高频小变量区 
    

 执行逻辑是： 

  1. 读到操作数 0x20 
  2. 把它当地址 0x0020 
  3. 去总线读这个地址的内容 
  4. 把读到的值给 A 

 所以： 

 LDA #$20 是把 0x20 当值` 

 这俩一定不能混。 

---------

7. ZPX / ZPY：零页加索引 


 比如： 

 LDA $20,X 

 意思不是“读 $20 和 X 两个地方”，而是： 

  1. 先取零页基地址 $20 
  2. 再加上 X 
  3. 得到最终地址 
  4. 去读数据 

 如果 X = 5，那最终地址就是： 

  - 0x20 + 0x05 = 0x25 
    

 这类模式很适合做数组、表、循环访问。 

 零页索引的一个要点是： 

 它通常在 8 位范围里绕回 

 也就是你以后会看到零页加法不是普通 16 位地址加法。 

 现在先记概念，不抠实现细节。 

---------

8. ABS：绝对寻址 


 ABS = Absolute 

 比如： 

 LDA $2002 

 这时候操作数不再是 1 字节，而是完整 16 位地址。

 流程是： 

  1. 从 ROM 再读两个字节 
  2. 拼成地址 0x2002 
  3. 去总线读这个地址 
  4. 把值加载到 A 

 这就是最普通的“给出完整地址”的方式。

---------

9. ABX / ABY：绝对地址加索引 


 比如： 

 LDA $2000,X 
 LDA $2000,Y 

 流程就是： 

  1. 读出基地址 0x2000 
  2. 加上 X 或 Y 
  3. 得到最终地址 
  4. 去读数据 
     

 这和零页加索引类似，只不过基地址是完整 16 位。 

 这种模式常用于： 

- 访问更大的数组/表 
- 图形数据查表 
- 循环处理块数据 


REL：相对寻址 


 REL = Relative 

 这个主要给分支指令用，比如： 

 BEQ label 

 它后面的操作数通常不是“绝对地址”，而是： 

 相对于当前 PC 的偏移量 

 也就是说： 

  - 如果偏移是正的，就往后跳 
  - 如果偏移是负的，就往前跳
    

 为什么这么设计？ 

 因为分支一般只需要在附近跳，不需要写完整 16 位地址。 

 你现在先记住： 

 REL 不是给出目标地址，而是给出“离当前位置差多少” 

11. IND：间接寻址 


 IND = Indirect 

 最经典的是： 

 JMP ($1234) 

 注意这里有括号。 

 它的意思不是： 

  - 直接跳去 0x1234 
    

 而是： 

  1. 先去读地址 0x1234 处的内容 
  2. 再把读到的那个 16 位值当成真正跳转目标 


 也就是： 

 操作数给的是“指针所在地址” 

 所以这是“地址的地址”。 

12. IZX：Indexed Indirect，(zp,X) 


 写法像这样： 

 LDA ($20,X) 

 这类很容易绕，我给你拆成步骤。 

  1. 先拿零页基地址 $20 
  2. 加上 X 
  3. 得到零页中的某个位置 
  4. 从这个零页位置开始，读出一个 16 位地址 
  5. 再去那个真正地址读数据 


 所以它是： 

 先用 X 去找一个指针，再沿着这个指针找数据 

 你可以把它叫做： 

 先索引，后间接 

 这就是 (zp,X) 这个名字的含义。 

13. IZY：Indirect Indexed，(zp),Y 


 写法像这样： 

 LDA ($20),Y 

 这和上一个顺序相反。 

  1. 先在零页地址 $20 处读出一个 16 位基地址 
  2. 再把 Y 加上去 
  3. 得到最终地址 
  4. 再去读数据


 所以它是： 

 先间接，后索引 

 也就是： 

  - (zp,X)：先加 X，再解引用 
  - (zp),Y：先解引用，再加 Y 
    

 这两个模式是初学者最容易混的地方之一。 

14. 一个最重要的对比 


 把下面这四条放一起看：

 LDA #\$20 
 LDA \$20 
 LDA (\$20,X) 
 LDA (\$20),Y 

 它们都叫 LDA，但意思完全不同：

  - #$20 
    直接把 0x20 当值 
  - $20 
    把 0x20 当零页地址 
  - ($20,X) 
    先在零页里找“指针位置”，再沿指针取数 
  - ($20),Y 
    先在零页里读出“基地址”，再加 Y 
    

 所以以后学指令时，不能只看 mnemonic，比如只看 LDA。 
 一定要把“指令 + 寻址模式”一起看。 

15. Rust 实现时该怎么想 


 你后面实现 CPU 时，很适合把寻址模式单独做成一层。 

 比如： 

```rust
enum AddrMode { 
    Imp, 
    Imm, 
    Zp0, 
    Zpx, 
    Zpy, 
    Rel, 
    Abs, 
    Abx, 
    Aby, 
    Ind, 
    Izx,
    Izy, 
}  
```

 然后为每种寻址模式写一个“解析操作数”的函数。 

 返回值常见有两种思路： 

  - 返回“最终地址” 
  - 或返回“取值所需的上下文” 
    

 比如： 

```rust
 fn resolve_operand_address(&mut self, mode: AddrMode) -> u16 
```

 不过不是所有模式都天然返回“最终地址”，比如 IMM 更像“值就在 PC 指向的位置”。 
 所以后面我们会再细化这个设计。 

 你现在先建立一个意识： 

 寻址模式最好独立建模，不要把所有逻辑糊进每条指令实现里。 


---------

  16. 这节课你必须记住的 6 句话 
      
      寻址模式回答的是“数据从哪里来”。 
      
      同一条指令配不同寻址模式，行为路径会很不同。 
      
      \# 表示立即数，后面是值本身。 
      
      零页寻址是把 8 位操作数当成 0x00xx 地址。 
      
      ABS 是完整 16 位地址，REL 是相对偏移。 
      
      (zp,X) 和 (zp),Y 的核心区别是“先索引还是先间接”。
