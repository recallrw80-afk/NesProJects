第 5 课：6502 CPU 的寄存器和状态位 

这节课的目标很明确： 

  1. 知道 CPU 内部到底保存哪些核心状态 
  2. 知道这些状态为什么足够支撑程序运行 
  3. 先不背指令，只理解“CPU 平时在改什么” 

CPU 模拟的本质就是维护这些寄存器状态。 
    1. CPU 本质上是一个状态机 

 你可以先把 CPU 粗暴理解成： 

```rust
struct Cpu {
    a: u8,
    x: u8,
    y: u8,
    sp: u8,
    pc: u16, 
    p: Status, 
}   
```

为什么说 CPU 是“状态机”？ 因为每执行一条指令，本质上就是： 

  - 读一点数据 
  - 改几个寄存器 
  - 改几个标志位 
  - 改 PC 
  - 写数据 

 所以以后你看到一条 6502 指令时，脑子里要自动变成这个问题： 这条指令会改 CPU 的哪些状态？ 
    2. 6 个最重要的寄存器 

  A：Accumulator，累加器 这是最常用的寄存器。 很多运算默认都围绕它展开。 比如： 
  - LDA：把值装进 A 
  - ADC：拿 A 做加法 
  - AND：拿 A 做按位与 
  - STA：把 A 写回总线地址

 你可以把它理解成 CPU 手边最顺手的工作寄存器。

 X、Y：索引寄存器 这两个寄存器常用于： 

  - 地址偏移 
  - 循环计数 
  - 暂存中间值 
  - 某些指令的数据搬运 

 为什么要有两个？ 因为 6502 的很多寻址模式都依赖 X 或 Y，比如： 
  - Zero Page,X 
  - Absolute,X 
  - (zp),Y 

 你现在先不用记寻址细节，只要知道： X/Y 经常参与“定位数据在哪”。 

PC：Program Counter，程序计数器 

这是非常关键的寄存器。 它表示： CPU 下一步要从哪个地址取指令 比如： 

  - PC = 0xC000 
  - 那 CPU 接下来就去读 0xC000 的字节，当作 opcode 

然后 PC 会往后移动。 如果遇到跳转、子程序调用、中断，PC 会被改到别处。 你可以把 PC 理解成： 当前执行位置指针 

SP：Stack Pointer，栈指针 

这个寄存器指向栈的位置。 6502 的栈固定在： 

  - 0x0100..0x01FF 

 SP 本身只有 8 位，所以它不是完整地址，而是这一页里的偏移量。 比如： 
  - SP = 0xFD 
  - 实际对应的栈地址就是 0x01FD 

 当前项目里压栈和弹栈也能看出来：core/cpu.c bus_write(0x0100 + __sp, data); 

所以以后你要记住： 6502 的栈页是固定的，SP 只是页内偏移。 

P：Processor Status，状态寄存器 

这是 CPU 的标志位集合。 它不是普通数值寄存器，而是“很多 1-bit 状态的打包”。 这也是后面 CPU 模拟最容易错的地方之一。 

状态寄存器 P 里有哪些位 

  - C：Carry 
  - Z：Zero 
  - I：Interrupt Disable 
  - D：Decimal 
  - B：Break 
  - U：Unused 
  - V：Overflow 
  - N：Negative 

 我们一个个讲，但只讲“够你现在用的版本”。 

C：Carry，进位/借位标志 常见于： 

  - 加法进位 
  - 减法借位 
  - 位移旋转 

 你现在先记： 无符号运算有没有溢出到 8 位外面，经常看 C 

Z：Zero，零标志

如果某次运算结果是 0，通常就会置 Z = 1。 比如： 

  - LDA #$00 后，Z = 1 
  - LDA #$01 后，Z = 0 

 N：Negative，负号标志 本质上通常是看结果的最高位 bit7。 
  - 如果结果 bit7 = 1，则 N = 1 
  - 否则 N = 0 

 比如： 
  - LDA #$80 后，N = 1 
  - LDA #$7F 后，N = 0 

 注意，它并不是真的“会算数学负号”，而是反映结果最高位。 

V：Overflow，溢出标志 

这个比较难，先只建立概念： V 主要描述“有符号运算结果是否异常溢出”。 它和 C 不一样。 

  - C 更偏无符号进位 
  - V 更偏有符号溢出 

 这两个后面学 ADC/SBC 时会重点讲，现在先知道不是一回事。

 I：Interrupt Disable，中断禁止标志 如果它被置位，CPU 会屏蔽一部分中断。 你现在先记： I 控制 CPU 是否响应普通 IRQ` 注意： 

  - NMI 叫“不可屏蔽中断”，一般不受它影响 
  - IRQ 通常会受它影响 

 D：Decimal，十进制模式 6502 原版有十进制模式，但 NES 使用的 2A03 CPU 实际上不真正支持十进制运算效果。 所以对 NES 模拟来说： 
  - 这个标志位仍然存在 
  - 但十进制算术通常不真正生效 

 你先只要知道“有这个位”，不用过度纠结。 

B：Break 跟 BRK 指令和中断压栈有关。 你现在先记成“与中断/BRK 过程有关的特殊标志”。

U：Unused 通常视为保留位。 很多实现里会让它保持为 1 或按约定处理。 你现在先不用深究。 

一个很直观的小例子 

 假设现在执行： LDA #$00 这条指令干了什么？ 
  - 把立即数 0x00 放进 A 
  - 因为结果是 0，所以 Z = 1 
  - 因为 bit7 不是 1，所以 N = 0 

 也就是说，执行后状态可能变成： a = 0x00 z = true n = false 再看： LDA #$80 执行后： 
  - A = 0x80 
  - Z = 0 
  - N = 1 

 这就是为什么我一直强调： 指令不只是改寄存器数值，还会改标志位。 
    5. 为什么标志位这么重要 

 因为后面很多分支跳转都靠它们。 比如： 
  - BEQ：如果 Z = 1 就跳 
  - BNE：如果 Z = 0 就跳 
  - BMI：如果 N = 1 就跳 
  - BCC：如果 C = 0 就跳 

 所以你可以把状态位看成： CPU 刚刚运算后留下的“结论标签” 后续指令会根据这些标签决定流程。 
    6. reset 时 CPU 从哪开始运行 

 这件事和 PC 直接相关。 CPU 上电或 reset 后，不是随便从某个地址开始执行。 它会去固定位置读一个“启动地址”，再把它装入 PC。 这个地址来源后面会讲到 reset vector，你现在先知道： PC 的初始值不是写死在代码里的，而是从内存特定位置读出来的。 这就是为什么 ROM 映射和启动向量很重要。 
    7. Rust 里状态寄存器怎么表示 

 你将来可以有两种常见写法。 一种是拆字段： 

```rust
  struct Status {
      c: bool,
      z: bool,
      i: bool,
      d: bool, 
      b: bool, 
      u: bool, 
      v: bool, 
      n: bool,  
  }
```

好处是好懂。 另一种是存成一个 u8，再写辅助函数： 

```rust
struct Cpu { p: u8, } 
```

比如：

```rust
 fn set_flag(&mut self, mask: u8, value: bool) { ... } 
 fn get_flag(&self, mask: u8) -> bool { ... } 
```

好处是更贴近真实寄存器，也更方便压栈/还原。 对你当前阶段，我建议： 先用你最容易读懂的方式实现，再慢慢优化表示法。 

    8. 这节课最核心的认知 

 现在你应该开始形成这个思维： CPU 并不是“执行一堆函数”那么简单， 而是维护下面这套最重要的运行状态： 
  - A/X/Y：工作数据 
  - PC：下一条指令在哪 
  - SP：栈现在在哪 
  - P：上一次运算和中断控制状态 

 以后每讲一条指令，你都可以用一句话拆解： 这条指令读了什么，改了哪些寄存器，改了哪些标志位，PC 怎么变。 如果你能这样想，CPU 就真正开始入门了。 
  9. 这节课你必须记住的 5 句话 

     CPU 模拟的核心是维护寄存器状态。 

     PC 决定下一条指令从哪里取。 

     SP 是固定栈页里的偏移，不是任意地址。 

     P 不是普通数值，而是一组标志位。 

     很多指令除了改数据，还会顺手改 Z/N/C/V 等状态位。
