第23课：PPU 的最小认知模型 

  - CPU 不直接画像素 
  - CPU 通过 PPU 寄存器“告诉 PPU 该怎么工作” 
  - PPU 自己按节奏去取图块、取属性、取精灵、取调色板 
  - 最后把画面一行一行扫出来 

 所以 PPU 不是 framebuffer，它更像一台“按规则生成画面”的图形芯片。 


  1. 先分清 PPU 里最重要的 5 类数据 
     

 Pattern Table 

  - 图块像素原始数据 
  - 你可以先把它理解成“字形库/贴图库” 
  - 背景和精灵显示时，最终都要从这里取图形位图 
    

 Name Table 

  - 背景地图 
  - 它不存真正像素，而是存“这个位置该用哪个图块编号” 
  - 可以理解成“地图格子表” 
    

 Attribute Table 

  - 背景调色信息 
  - Name Table 决定“放哪个砖块” 
  - Attribute Table 决定“这片区域用哪组颜色” 
    

 Palette 

  - 调色板 
  - 决定最终颜色索引对应什么颜色 
    

 OAM 

  - 精灵表 
  - 存每个精灵的位置、图块号、属性等 
  - 你这次写的 DMA，就是在往这里搬 256 字节数据 
    

  2. 先把背景渲染理解成两层 
     

 你不要一开始就想“屏幕像素怎么来”，先想两层： 

 第一层：地图层 

  - 屏幕某个区域该显示哪个 tile 
  - 这来自 Name Table 
    

 第二层：图形层 

  - 这个 tile 的真正像素长什么样 
  - 这来自 Pattern Table 
    

 然后再叠加颜色： 

  - 颜色来自 Attribute Table + Palette 
    

 所以背景输出不是“直接存好像素”，而是现场查表组合出来的。 

  3. 精灵是什么 
     

 精灵可以先理解成： 

  - 一组单独摆放在画面上的小对象 
  - 比如人物、子弹、敌人 
    

 它们的信息放在 OAM 里，典型包含： 

  - Y 坐标 
  - tile 编号 
  - 属性 
  - X 坐标 
    

 这也是为什么 OAM DMA 很重要： 

  - 游戏每帧都可能要大批量更新精灵 
  - 一次搬 256 字节，比 CPU 自己慢慢写方便得多 
    

  4. PPU 为什么一定需要寄存器 
     

 因为 CPU 看不到 PPU 内部细节。 
 CPU 只能通过“门口的几个控制按钮”去影响它。 

 这些“按钮”就是 PPU 寄存器。 

 你当前已经碰到最早的一组： 

  - $2003：OAMADDR 
  - $2004：OAMDATA 
    

 它们说明了一件事： 

  - CPU 不是直接访问 ppu.oam[index] 
  - 而是通过寄存器协议和 PPU 交互 
    

 以后你还会遇到更重要的： 

  - $2000 PPUCTRL 
  - $2001 PPUMASK 
  - $2002 PPUSTATUS 
  - $2005 PPUSCROLL 
  - $2006 PPUADDR 
  - $2007 PPUDATA 
    

 这些是完整 PPU 的真正入口。 

  5. 为什么 \$2006/\$2007 很重要 
     

 这是理解 PPU 不是普通内存的关键。

 CPU 访问普通 RAM 是： 

  - 直接给地址 
  - 直接读写 
    

 但 CPU 访问 PPU 内部 VRAM 时，不是这样。 
 它通常要通过寄存器间接访问： 

  1. 先往 $2006 写地址 
  2. 再通过 $2007 读写数据 
     

 这说明： 

  - PPU 有自己的地址空间 
  - CPU 是通过寄存器窗口去访问它 
  - 这不是普通内存映射的直读直写模型 
    

  6. 你现在最该建立的 PPU 脑图 
     

 你先把 PPU 想成： 

  - 一边有寄存器接口给 CPU 用 
  - 一边有自己内部的数据区 
  - 一边按自己的节奏扫描输出画面 
    

 也就是： 

 CPU <-> PPU registers <-> PPU internal memory/state -> video output 

 这比“PPU 就是一块显存”准确得多。 

  7. 当前你写的最小 PPU 在整个课程里的位置 
     

 你现在 D:/code/rust/nes_core/src/ppu.rs 的作用不是“已经完成图形芯片”，而是： 

  - 先把 OAM 这个独立概念立住 
  - 让 DMA 有目标可以写 
  - 让你从 CPU 视角看到“PPU 是一个带寄存器接口的设备” 
    

 这一步非常对，别急着一次把 PPU 全写完。 

  8. 你接下来应该怎么学 PPU 
     

 正确顺序不是“直接渲染画面”，而是： 

  1. 先认清寄存器和内部数据区分别是什么 
  2. 再理解 PPU 地址空间和镜像 
  3. 再理解滚动、背景取数、精灵取数 
  4. 最后才是扫描线和逐点输出 
     

 否则很容易被细节淹没。 

  9. 这节课最关键的结论 
     
  - Pattern Table：图块图形库 
  - Name Table：背景地图 
  - Attribute Table：背景颜色区域信息 
  - Palette：最终颜色表 
  - OAM：精灵信息表 
    

 以及： 

  - CPU 不直接画像素 
  - CPU 通过寄存器控制 PPU 
  - PPU 按自己的时序组合这些数据并输出画面
