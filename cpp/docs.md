## 一、什么是 NES？

NES（Nintendo Entertainment System，任天堂娱乐系统）是任天堂 1983 年发布的 8 位游戏机。它使用一颗 **Ricoh 2A03** CPU（基于 MOS 6502，但去掉了 BCD 模式，增加了音频处理），以及一颗独立的 **PPU**（图像处理单元）。

### NES 的关键硬件规格

| 组件 | 规格 |
|---|---|
| **CPU** | 8 位 6502 定制版，主频 ~1.79 MHz（NTSC）|
| **RAM** | 2 KB 主内存（CPU 用）|
| **VRAM** | 2 KB（PPU 用，存命名表和属性表）|
| **卡带** | 程序 ROM (PRG-ROM) + 图形 ROM (CHR-ROM) |
| **分辨率** | 256×240 像素 |
| **颜色** | 系统调色板 64 色，屏幕同时显示 25 色 |
| **音频** | 5 个通道（2 方波 + 1 三角波 + 1 噪声 + 1 DMC 采样）|

**核心思想：总线架构 (Bus)**

NES 中所有组件通过 **总线 (Bus)** 通信。CPU 不直接访问内存或卡带——它通过总线读写地址。总线负责**路由**：看到某个地址范围，就知道该发给谁。

## 四、逐组件深度讲解

### 4.1 类型定义 (`types.h`)

```cpp
using u8  = std::uint8_t;   // 无符号 8 位  (0~255)
```

### 4.2 卡带 (`Cartridge`)

**NES 卡带是什么？**

物理上是一块插在机器上的电路板，里面装着 ROM 芯片。模拟器需要读取 `.nes` 文件来模拟卡带。

#### iNES 文件格式（.nes）

这是最通用的 NES ROM 文件格式，你的 `Cartridge::load()` 已经在解析它：

```
偏移量  大小    含义
──────────────────────────────────────
0       4       Magic: "NES" + 0x1A (字节: 4E 45 53 1A)
4       1       PRG-ROM 的 16KB bank 数量
5       1       CHR-ROM 的 8KB bank 数量
6       1       Flags 6 (包含 Mapper 低4位、镜像、电池RAM、Trainer)
7       1       Flags 7 (包含 Mapper 高4位)
8       8       保留/零填充
16      ...     Trainer（如果存在，512字节）
...     ...     PRG-ROM 数据 (prg_banks * 16384 字节)
...     ...     CHR-ROM 数据 (chr_banks * 8192 字节)
```

**你已经实现的：**
- ✅ 读取文件到内存
- ✅ 验证 magic number (`NES` + `0x1A`)
- ✅ 解析 PRG/CHR bank 数量
- ✅ 解析 Mapper ID
- ✅ 解析镜像模式、电池 RAM、Trainer 标志
- ✅ 打印卡带信息

**还需要做的：**
- ❌ 真正读取 PRG-ROM 和 CHR-ROM 数据存储到 `prg_rom` 和 `chr_rom` 向量中
- ❌ 实现 `cpu_read()` / `cpu_write()` 和 `ppu_read()` / `ppu_write()` —— 这些要委托给 Mapper

### 4.3 Mapper（存储体切换器）—— NES 最复杂的部分

**为什么需要 Mapper？**

NES 的 CPU 只有 16 位地址线，可寻址范围是 64 KB。但游戏卡带可能有 256KB 甚至更大的 PRG-ROM。怎么让 CPU 访问到超出 64KB 的 ROM 呢？答案就是 **Mapper**。

Mapper 是一种**存储体切换（bank switching）**硬件，位于卡带上。它截获 CPU 的地址请求，把它映射到 ROM 的实际物理地址。

```
CPU 发出地址 0x8000
        │
        ▼
   ┌─────────┐
   │ Mapper  │  ← 看到 0x8000 落在 PRG-ROM 区域
   │         │     "哦，当前 bank 0 被映射到物理 0x00000"
   │ bank 0  │     → 返回 PRG-ROM[0x00000 + (0x8000-0x8000)]
   └─────────┘
```

NES 有几百种 Mapper。最重要、最常见的是：

| Mapper | 代表游戏 | 复杂度 |
|---|---|---|
| **Mapper 0 (NROM)** | 超级马里奥兄弟、大金刚 | 最简单，无 bank switching |
| **Mapper 1 (MMC1)** | 塞尔达传说、银河战士 | 中等 |
| **Mapper 2 (UxROM)** | 恶魔城、洛克人 | 简单 |
| **Mapper 3 (CNROM)** | 某些早期游戏 | 简单 |
| **Mapper 4 (MMC3)** | 超级马里奥兄弟 3、星之卡比 | 复杂，需要扫描线中断 |

**架构设计建议：**

```
        ┌─────────────┐
        │   Mapper    │  ← 抽象基类（接口）
        │ (虚基类)    │
        └──────┬──────┘
               │
    ┌──────────┼──────────┐
    │          │          │
    ▼          ▼          ▼
Mapper0    Mapper1    Mapper4
(NROM)    (MMC1)     (MMC3)
```

基类 `Mapper` 定义接口：
- `cpu_map_read(addr)` → 映射后的地址
- `cpu_map_write(addr, data)` → 处理写操作（可能触发 bank switch）
- `ppu_map_read(addr)`
- `ppu_map_write(addr, data)`

**Mapper 0 (NROM) 是最简单的：**
- 如果 PRG-ROM 只有 16KB（1 bank），地址 `0x8000-0xBFFF` 和 `0xC000-0xFFFF` 映射到**同一块**（镜像）
- 如果 PRG-ROM 是 32KB（2 banks），各映射 16KB，无需 bank switching
- CHR-ROM 直接映射，无需切换

### 4.4 CPU —— 模拟器的心脏

**6502 CPU 核心知识：**

#### 寄存器（6个）
```
┌──────────────────────────────────────┐
│  A  (累加器)   8位   算术/逻辑运算     │
│  X  (X变址)    8位   寻址用           │
│  Y  (Y变址)    8位   寻址用           │
│  PC (程序计数器) 16位  当前指令地址     │
│  SP (栈指针)    8位   栈顶 (0x0100+SP)│
│  P  (状态寄存器) 8位   标志位          │
└──────────────────────────────────────┘
```

#### 状态寄存器 P 的标志位
```
7  6  5  4  3  2  1  0
N  V  -  B  D  I  Z  C
│  │     │  │  │  │  │
│  │     │  │  │  │  └── C: 进位标志 (Carry)
│  │     │  │  │  └──── Z: 零标志 (Zero)
│  │     │  │  └─────── I: 中断禁用 (Interrupt Disable)
│  │     │  └────────── D: 十进制模式 (NES 6502 无此模式)
│  │     └───────────── B: 软件中断标志 (Break)
│  └─────────────────── V: 溢出标志 (Overflow)
└───────────────────── N: 负数标志 (Negative)
```

#### CPU 内存映射（最重要的地址布局）

这是 CPU 看到的 64KB 地址空间（**不是实际内存**，是映射视图）：

```
地址范围        大小     内容
────────────────────────────────────────────
0x0000 - 0x07FF  2KB    CPU 内部 RAM（真正的工作内存）
0x0800 - 0x1FFF  镜像   上面 2KB 的镜像（重复 3 次）
0x2000 - 0x2007  8字节   PPU 寄存器（控制 PPU 的窗口！）
0x2008 - 0x3FFF  镜像   PPU 寄存器的镜像
0x4000 - 0x4017  18字节  APU 和 I/O 寄存器
0x4018 - 0x401F  禁用    （测试用）
0x4020 - 0xFFFF  剩余   卡带 PRG-ROM / 卡带 RAM
    ├─ 0x6000-0x7FFF: 卡带 SRAM（存档用，电池供电）
    └─ 0x8000-0xFFFF: PRG-ROM（程序代码和数据）
```

#### 中断向量（6502 启动时读取这里）
```
地址         内容
──────────────────────────
0xFFFA-0xFFFB  NMI 中断向量（PPU VBlank时触发）
0xFFFC-0xFFFD  复位向量（机器启动时跳转到这里）
0xFFFE-0xFFFF  IRQ/BRK 中断向量
```

#### 6502 指令集概览

6502 有 56 条官方指令，加上不同的**寻址模式**，共有约 151 个操作码。

主要分类：
- **数据传输**：`LDA`（加载到A）、`STA`（存储A）、`LDX`、`STX`、`LDY`、`STY`
- **算术运算**：`ADC`（带进位加）、`SBC`（带借位减）、`INC`、`DEC`
- **逻辑运算**：`AND`、`ORA`、`EOR`、`LSR`、`ASL`、`ROL`、`ROR`
- **比较**：`CMP`、`CPX`、`CPY`、`BIT`
- **跳转**：`JMP`、`JSR`（跳转子程序）、`RTS`（返回）、`RTI`（中断返回）
- **分支**：`BEQ`、`BNE`、`BCC`、`BCS`、`BPL`、`BMI`、`BVC`、`BVS`
- **栈操作**：`PHA`、`PLA`、`PHP`、`PLP`、`TXS`、`TSX`
- **系统**：`NOP`、`BRK`、`CLC`、`SEC`、`CLI`、`SEI`、`CLV`、`CLD`、`SED`

#### 寻址模式（CPU 如何找到操作数）

这是 6502 最需要理解的部分：

| 模式 | 格式 | 含义 |
|---|---|---|
| **隐含** | `TAX` | 操作数隐含在指令中 |
| **累加器** | `ASL A` | 操作 A 寄存器 |
| **立即数** | `LDA #$10` | 操作数是指令的一部分 |
| **零页** | `LDA $10` | 从地址 0x0010 读取（零页：0x0000-0x00FF）|
| **零页,X** | `LDA $10,X` | 地址 = 0x0010 + X（在零页内回绕）|
| **零页,Y** | `LDX $10,Y` | 地址 = 0x0010 + Y |
| **绝对** | `LDA $2000` | 从完整 16 位地址读取 |
| **绝对,X** | `LDA $2000,X` | 地址 = 0x2000 + X |
| **绝对,Y** | `LDA $2000,Y` | 地址 = 0x2000 + Y |
| **间接,X** | `LDA ($10,X)` | 地址 = 内存[0x0010 + X]（零页内）|
| **间接,Y** | `LDA ($10),Y` | 地址 = 内存[0x0010] + Y |
| **相对** | `BEQ $+10` | PC = PC + 有符号偏移量（分支指令用）|

#### CPU 执行周期

每条指令的执行流程：
```
1. 取指 (Fetch)     → 从 PC 地址读取操作码
2. 译码 (Decode)    → 查表：这是什么指令？什么寻址模式？
3. 取操作数          → 根据寻址模式读取数据
4. 执行             → 真正执行操作
5. 写回             → 将结果写回目标地址
6. 更新 PC          → 指向下一条指令
```

CPU 实现的核心就是一个大循环：
```
while (运行中) {
    操作码 = 读内存(PC);
    PC++;
    根据操作码查表 → 执行对应的指令函数;
    更新周期计数;
}
```

### 4.5 PPU（图像处理单元）—— 最复杂的部分

PPU 是 NES 模拟中最难但最有成就感的部分。它独立于 CPU 运行，有自己的地址空间。

#### PPU 渲染原理（从 PPU 视角）

NES 的屏幕是 **256×240 像素**。但 PPU 内部渲染 **256×240 像素** 的画面，通过以下结构：

```
屏幕 = 背景 (Background) + 精灵 (Sprites)

背景渲染过程：
1. 从命名表 (Nametable) 获取 tile 索引
2. 从图案表 (Pattern Table) 获取 tile 的像素数据
3. 从属性表 (Attribute Table) 获取调色板编号
4. 从调色板获取最终 RGB 颜色
```

#### PPU 内存映射

```
地址范围          大小     内容
────────────────────────────────────────────
0x0000 - 0x0FFF   4KB    图案表 0 (Pattern Table 0) —— 来自 CHR-ROM
0x1000 - 0x1FFF   4KB    图案表 1 (Pattern Table 1) —— 来自 CHR-ROM
0x2000 - 0x23FF   1KB    命名表 0 (Nametable 0) —— VRAM 中
0x2400 - 0x27FF   1KB    命名表 1 (Nametable 1)
0x2800 - 0x2BFF   1KB    命名表 2 (Nametable 2)
0x2C00 - 0x2FFF   1KB    命名表 3 (Nametable 3)
0x3000 - 0x3EFF   镜像   上面 0x2000-0x2EFF 的镜像
0x3F00 - 0x3F1F   32字节 调色板 (Palette) —— 背景 16 + 精灵 16
0x3F20 - 0x3FFF   镜像   调色板的镜像
```

#### PPU 寄存器（CPU 通过 0x2000-0x2007 访问）

```
CPU地址   PPU寄存器    功能
──────────────────────────────────────────
0x2000   PPUCTRL     控制寄存器（NMI使能、命名表选择、VRAM增量等）
0x2001   PPUMASK     遮罩寄存器（控制哪些图层显示）
0x2002   PPUSTATUS   状态寄存器（VBlank标志、精灵溢出等）
0x2003   OAMADDR     OAM地址（精灵属性内存的写入位置）
0x2004   OAMDATA     OAM数据（写入精灵属性）
0x2005   PPUSCROLL   滚动寄存器（设置背景滚动偏移）
0x2006   PPUADDR     PPU地址寄存器（设置读写地址）
0x2007   PPUDATA     PPU数据（读写VRAM/调色板数据）
```

#### Tile 和 命名表

NES 的最小图形单位是 **Tile（图块）**，每个 tile 是 **8×8 像素**。

**命名表**是一个 32×30 的 tile 索引数组（1024 字节 = 32×30 = 960 个 tile 索引 + 64 字节属性表）。每个字节告诉 PPU："在这个位置画图案表的第几个 tile"。

```
屏幕 = 32 × 30 个 tiles = 1024 字节命名表
命名表每个字节 = tile 索引（指向图案表的哪个 8×8 tile）
属性表每个字节 = 4 个 tile 的颜色组编号（2×2 tile 区域）
```

**属性表**：每字节控制 4 个 tile（2×2 区域）的颜色调色板编号，因为 1 字节 = 8 位，每个区域只能用 2 位（4 种选择之一）。

#### 精灵 (Sprites)

精灵是独立于背景的**可移动图形元素**（马里奥、敌人、子弹等）。

- NES 有 64 个精灵
- 每个精灵占用 OAM（Object Attribute Memory）中的 4 个字节：
  - 字节 0: Y 坐标
  - 字节 1: Tile 索引
  - 字节 2: 属性（调色板、水平/垂直翻转、优先级）
  - 字节 3: X 坐标
- OAM 共 256 字节（64 × 4）

#### 渲染过程（逐行扫描）

NES 的 PPU 按扫描线渲染，而不是一整帧：

```
每一帧 = 262 条扫描线 (NTSC)
  ├─ 扫描线 0-239：可见画面（240 行）
  ├─ 扫描线 240：哑扫描线（不渲染）
  ├─ 扫描线 241-260：VBlank 期间（20 行）
  │    └─ 在 241 行开始时触发 NMI 中断
  └─ 扫描线 261：预渲染扫描线

每条扫描线 = 341 个 PPU 周期 (dots)
  ├─ 周期 0-255：取 tile 数据并渲染 256 个像素
  ├─ 周期 256-320：取精灵数据
  └─ 周期 321-340：为下一条扫描线做预取
```

**VBlank 是关键：** 当扫描线 241 开始时，PPU 触发 NMI（不可屏蔽中断），告诉 CPU "一帧画完了，可以更新数据了"。游戏逻辑通常在 NMI 中运行。

#### 镜像 (Mirroring)

NES 只有 2KB VRAM 用于命名表。但逻辑上需要 4 个命名表（4KB）。解决方案是**镜像**：

```
水平镜像 (Horizontal)：
  命名表 0 ←→ 命名表 1 (水平方向镜像)
  命名表 2 ←→ 命名表 3

垂直镜像 (Vertical)：
  命名表 0 ←→ 命名表 2 (垂直方向镜像)
  命名表 1 ←→ 命名表 3
```

### 4.6 APU（音频处理单元）

NES 有 5 个音频通道：

| 通道 | 类型 | 特点 |
|---|---|---|
| **Pulse 1** | 方波 | 4 种占空比（12.5%/25%/50%/75%），有音量包络和扫频 |
| **Pulse 2** | 方波 | 同上，无扫频 |
| **Triangle** | 三角波 | 低音好，无音量控制（固定音量）|
| **Noise** | 伪随机噪声 | 用于鼓声、音效、爆炸声 |
| **DMC** | Δ 调制采样 | 可播放小段采样音频（如马里奥的 "咚咚" 跳声）|

APU 通过 CPU 的 `0x4000-0x4017` 地址范围控制。

音频输出是这 5 个通道的**混合信号**。模拟器需要以 44100Hz 或 48000Hz 采样率输出 PCM 音频。

### 4.7 总线 (Bus)

**总线是粘合剂。** CPU 不直接知道内存、PPU、APU 在哪里——它只是通过总线读写地址。

```cpp
// 总线的工作逻辑（伪代码）
u8 Bus::cpu_read(u16 address) {
    if (address < 0x2000)         → 从 CPU RAM 读取
    else if (address < 0x4000)    → 从 PPU 寄存器读取
    else if (address < 0x4018)    → 从 APU/IO 寄存器读取
    else if (address >= 0x4020)   → 交给卡带/Mapper 处理
}
```

**DMA（直接内存访问）**：当 CPU 向 `0x4014` 写入一个值时，触发 DMA——256 字节的数据从 CPU 内存复制到 PPU 的 OAM。这发生在 CPU 写入 `0x4014` 后的 513-514 个 CPU 周期内。

## 六、你现在最应该理解的三个核心概念

### 1. 地址映射的思想

NES 的一切都是地址。CPU 不知道什么是 "PPU"——它只知道"往 `0x2006` 写个值，PPU 的地址寄存器就被设置了"。理解地址映射就理解了 NES 架构的一半。

### 2. 时序同步

CPU 和 PPU **并行运行**。PPU 按照自己的节奏（每周期 3 个 PPU dots）运行。模拟器通常采用"CPU 先跑一批周期，PPU 追赶"的策略来保持同步。

```
CPU:PPU 频率比
NTSC: CPU 1.789773 MHz, PPU 5.369318 MHz
比例 = 1:3（1 个 CPU 周期 = 3 个 PPU 周期）
```

### 3. 测试驱动

NES 模拟器**不能靠猜**。社区有大量现成的测试 ROM：
- **nestest** — 测试 CPU 指令正确性
- **Blargg's tests** — 全套 CPU/PPU/APU 测试
- **测试驱动开发是写模拟器的唯一正确方式**
