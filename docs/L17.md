17课：BRK / RTI 与中断向量（IRQ/NMI）

这节是 CPU 从“顺序执行”走向“被硬件打断后再返回”的关键课。 

先记住核心流程（BRK 和 IRQ/NMI 共用大框架）： 

1. 把当前执行位置压栈（PC 高字节、PC 低字节） 
2. 把状态寄存器压栈（P，带特定位语义） 
3. 设置 I=1（屏蔽普通 IRQ） 
4. 从中断向量读新 PC 并跳转 

---------

 RTI 就是反操作： 
  1. 从栈弹出 P 
  2. 从栈弹出 PC low 
  3. 从栈弹出 PC high 
  4. 继续执行被打断前的位置 

---------

  1) 三个向量地址（必须背下来） 
     
  - NMI vector: 0xFFFA/0xFFFB 
  - RESET vector: 0xFFFC/0xFFFD 
  - IRQ/BRK vector: 0xFFFE/0xFFFF 
    

 你这节主要实现 BRK 和 RTI，所以先用到 IRQ/BRK vector。 

---------

  2) BRK 的语义重点 
     

 BRK 是 1 字节指令，但 6502 会把它当成“触发一次软件中断”： 

  - 执行时，压栈的是“下一条地址”（等价于 PC 已越过 BRK 后的位置） 
  - 压栈状态字节里，B 位应表现为 1（只影响被压栈的字节表示） 
  - 然后跳转到 FFFE/FFFF 指定地址 
    

 你现在的简化实现可先按这个目标做，不急着 cycle-perfect。 

---------

  3) RTI 的语义重点 
     
  - RTI 会恢复 P 和 PC，不同于 RTS 
  - RTS 是给 JSR 用的，会 +1
  - RTI 不做 +1，直接恢复中断前保存的 PC 
    

 这点非常容易和 RTS 混淆。

 在 D:/code/rust/nes_core/src/cpu/mod.rs 里加两个函数（若已有就校验语义）：

 brk 伪代码： 

```rust
fn brk(&mut self, bus: &mut impl CPUBus) { 
    // BRK 触发时，PC 指向“下一字节之后”的语义你可以先按当前取指流程校准 
    self.stack_push((self.pc >> 8) as u8, bus); 
    self.stack_push((self.pc & 0xFF) as u8, bus); 
    let p = self.status_byte_for_push(true); // B=1 的压栈视图 
    self.stack_push(p, bus); 

    self.p.i = true; 
    self.pc = bus.cpu_read_u16(0xFFFE); 
} 
```
 rti 伪代码： 

```rust
fn rti(&mut self, bus: &mut impl CPUBus) { 
    let p = self.stack_pop(bus); 
    self.set_byte_to_p(p); 
    let lo = self.stack_pop(bus) as u16; 
    let hi = self.stack_pop(bus) as u16; 
    self.pc = (hi << 8) | lo; 
} 
```
 并在 exe_inst 里接： 

  - OpCode::BRK => self.brk(bus) 
  - OpCode::RTI => self.rti(bus)
