第 14 课：为什么真实 NES 程序还跑不起来，接下来该补什么 
 你现在可能会有一个感觉： 
 “我已经实现了很多 CPU 指令，也有很多测试，为什么还不能说 CPU 就完成了？” 
 答案是： 
 你现在实现的是一条很好的最小 CPU 主线，但离“能跑真实 ROM”还差几个大块。 
 先讲结论，接下来最重要的是这 4 组： 

  1. 更多分支和 flag 指令 
      - BCC/BCS/BMI/BPL/BVC/BVS 
      - 这些相对简单，能继续完善流程控制 
  2. 移位和旋转 
      - ASL/LSR/ROL/ROR 
      - 这是 CPU 核心能力之一，很多程序会用到 
      - 同时会真正开始大量使用 C 
  3. 内存自增自减与 bit test 
      - INC/DEC 
      - BIT 
      - 这些你已经开始碰到一部分了 
  4. 中断相关 
      - BRK 
      - RTI 
      - 最终要接 NMI/IRQ 
      - 但这个应该排在你把 CPU 指令基础补得更完整之后 

 最合理的下一步，不是立刻去 PPU 
 我建议你现在不要跳去 PPU。 
 理由很简单： 

  - PPU 比 CPU 更难 
  - CPU 还没到“足够支持真实程序跑初始化逻辑”的程度 
  - 如果现在跳去 PPU，你会同时面对两个大系统，调试会崩 

 所以正确顺序还是： 
 先把 CPU 再补一大截，再去接 Cartridge/真实 Bus/最小 ROM 执行 
 我建议你下一组实现目标 
 下一组优先补： 

  - BCC 
  - BCS 
  - BMI 
  - BPL 
  - BVC 
  - BVS 
  - ASL 
  - LSR 
  - ROL 
  - ROR 
  - BIT 

 为什么是这组： 
  - 分支组和你现在已有的 BEQ/BNE 逻辑高度类似 
  - 移位旋转会真正训练你对 C/Z/N 的理解 
  - BIT 会让你开始接触“不是所有指令都只是算值”的 flag 语义 

 你现在的工程阶段定位 
 如果按课程进度说，你现在已经从“纯概念课”进入了“CPU 实作中段”。 
 也就是说，你现在最需要的是： 

  - 一边实现 
  - 一边小步测试 
  - 一边补课堂里真正会影响实现决策的部分 

 所以后面我不会每次都讲很大的课，而会更偏： 
  - 讲一组指令背后的关键概念 
  - 让你立刻落代码 
  - 再 review
