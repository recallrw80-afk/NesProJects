第 13 课：ADC / SBC，Carry 和 Overflow 到底是什么

 这是 6502 里最容易混的一课。 
 几乎所有初学者都会在这里卡住，主要因为：

  - C 看起来像“进位” 
  - V 看起来也像“溢出” 
  - SBC 又把 C 的含义弄得更绕 
    

 所以这节课你先不要急着写代码，先把概念彻底理顺。 

---------

1. ADC 是什么 

 ADC = ADd with Carry 

 也就是： 

 A = A + M + C 

 这里的 C 是当前 carry flag，参与运算。 

 这和很多高级语言里的普通加法不一样。 
 它不是简单： 

 a + m 

 而是： 

 a + m + carry_in 

 其中： 

  - 如果 C = 0，就是普通加法 
  - 如果 C = 1，就是“再多加 1” 
    

 所以 C 在 ADC 里像“进位输入”。 

---------

2. C 在 ADC 里是什么意思 
   

 对于 ADC： 

 C 表示无符号加法有没有进位溢出到 8 位之外

 举个例子： 

 0xFF + 0x01 = 0x100 

 8 位结果只能留下： 

  - 0x00 
    

 多出来那一位就让： 

  - C = 1 
    

 所以： 

  - 结果寄存器里放低 8 位 
  - C 记录“有没有溢出去” 
    

 这叫无符号进位。 

---------

3. V 又是什么 
   

 V = Overflow 

 它不是和 C 重复，而是回答另一个问题： 

 如果把这些字节当有符号数看，这次结果有没有超出范围？ 

 8 位有符号数范围是： 

  - -128 .. 127 

 比如： 

 0x7F + 0x01 = 0x80 

 如果按无符号看： 

  - 127 + 1 = 128 
  - 没超过 255 
  - 所以 C = 0 
    

 但按有符号看： 

  - 127 + 1 
  - 应该得到 128 
  - 但 8 位有符号里没有 128 
  - 结果变成 -128 的编码 0x80 
    

 这就说明： 

  - V = 1 
    

 所以： 

 C 看无符号进位

 这两个一定要分开。 

---------

4. 一个最关键的对照例子 
   

 先记住这两个例子。 

 例子 1 

 0xFF + 0x01 = 0x00, C=1, V=0 

 为什么？ 

  - 无符号：255 + 1 超过 255，所以 C=1 
  - 有符号：-1 + 1 = 0，没有溢出，所以 V=0 
    

 例子 2 

 0x7F + 0x01 = 0x80, C=0, V=1 

 为什么？ 

  - 无符号：127 + 1 = 128，没有超过 255，所以 C=0 
  - 有符号：127 + 1 超出有符号上限，所以 V=1 
    

 这两个例子如果你真理解了，C 和 V 的区别就顺了大半。 

---------

5. ADC 要更新哪些 flag 
   

 最常见的是： 

  - C
  - Z 
  - V 
  - N 

 也就是： 

  - C：无符号进位 
  - Z：结果是否为 0 
  - V：有符号溢出 
  - N：结果 bit7 
    

 所以 ADC 是你目前碰到的第一条“需要认真更新多个 flag”的指令。 

---------

6. Rust 里 ADC 的最小实现思路 
   

 最朴素思路是先用更宽的类型算： 

```rust
fn adc(&mut self, addr: u16, bus: &mut impl CPUBus) { 
    let value = bus.cpu_read(addr); 
    let carry_in = if self.p.c { 1 } else { 0 }; 

    let sum = self.a as u16 + value as u16 + carry_in as u16; 
    let result = sum as u8; 

    self.p.c = sum > 0xFF; 
    self.p.z = result == 0; 
    self.p.n = (result & 0x80) != 0; 

    self.p.v = ((self.a ^ result) & (value ^ result) & 0x80) != 0; 

    self.a = result; 
} 
```

 这里最不好懂的是 V 的公式。 
 你现在不用先背公式，先懂语义： 

 两个同号数相加，结果却变成异号时，V=1 

 比如： 

  - 正 + 正 -> 负 
  - 负 + 负 -> 正 
    

 这就是有符号溢出。 

---------

7. SBC 为什么更绕 
   

 SBC = SuBtract with Carry 

 名字看着像减法，但里面仍然带 Carry。 

 最容易绕的点是： 

 在 SBC 里，C 不是“进位输入”，而更像： 

 没有借位 

 也就是说： 

  - C = 1：这次减法“正常减” 
  - C = 0：这次减法还要再额外减 1 
    

 所以： 

 A = A - M - (1 - C) 

 这才是 SBC 真正的语义。 

 如果： 

  - C = 1 
    那就是 A - M 

 如果： 

  - C = 0 
    那就是 A - M - 1 
    

 这就是它最绕的地方。 

---------

8. 为什么 SBC 的 C 表示“没有借位” 
   

 因为 6502 把减法做成了加法的补码形式。 
 底层实现细节现在不用深挖，但结果上你要记： 

 对于 SBC： 

  - C = 1 表示无借位 
  - C = 0 表示发生借位 
    

 也就是它和很多人直觉相反。 

 所以： 

 ADC 里 C 像“进位输入” 
 SBC 里 C 像“无借位标志” 

---------

9. SBC 的实现其实可以复用 ADC 思路 
   

 一个常见技巧是： 

 A - M - (1 - C) 

 等价于： 

 A + (~M) + C 

 也就是：

  - 把要减的数按位取反 
  - 再走一次 ADC 逻辑 
    

 所以很多模拟器会让 SBC 复用 ADC 的核心。 

 例如： 

```rust
fn sbc(&mut self, addr: u16, bus: &mut impl CPUBus) { 
    let value = bus.cpu_read(addr) ^ 0xFF; 
    let carry_in = if self.p.c { 1 } else { 0 };  
    let sum = self.a as u16 + value as u16 + carry_in as u16; 
    let result = sum as u8; 

    self.p.c = sum > 0xFF; 
    self.p.z = result == 0; 
    self.p.n = (result & 0x80) != 0; 
    self.p.v = ((self.a ^ result) & (value ^ result) & 0x80) != 0; 
    
    self.a = result; 
} 
```
 你以后可能会把 ADC/SBC 的共性再抽成一个辅助函数。 

---------

10. 先别被 V 公式吓住 
    

 你现在写代码时，优先做到： 

  - C 对 
  - Z 对 
  - N 对 
  - 基础 V 案例对 
    

 测试建议先锁住典型样例，而不是一开始就追求脑内完全推导公式。 

 比如先测： 

 ADC 

  1. 0x01 + 0x01 -> 0x02, C=0, V=0 
  2. 0xFF + 0x01 -> 0x00, C=1, Z=1, V=0 
  3. 0x7F + 0x01 -> 0x80, C=0, V=1, N=1 
     

 SBC

  1. 0x05 - 0x01，在 C=1 时结果应为 0x04 
  2. 0x00 - 0x01，在 C=1 时结果应借位，C=0 
  3. 0x00 - 0x00，在 C=1 时结果为 0x00，Z=1 
     
---------

11. CLC 和 SEC 为什么会变重要 
    

 前面你也实现了它们，但现在它们真正有用了。 

 因为： 

  - CLC：清除 C 
  - SEC：设置 C 
    

 而 C 会直接影响： 

  - ADC 
  - SBC 
  - 分支 BCC/BCS 
    

 所以从现在开始，Carry 不再只是个旁观 flag，而是运算输入的一部分。 

---------

12. 你这节课最该记住的 6 句话 
13. 

ADC 计算的是 A + M + C。 

C 在 ADC 里表示无符号进位。 

V 在 ADC/SBC 里表示有符号溢出。 

SBC 计算的是 A - M - (1 - C)。 

C 在 SBC 里表示“没有借位”。 

C 和 V 不是一回事，一个看无符号，一个看有符号。
