第16课：BIT 指令，为什么“读了但不写回，也不改 A”依然很重要

 这节课我们讲 BIT。 
 它看起来很怪，因为它不像 AND/ORA/EOR 那样把结果写回 A，但它在 6502 里很有用，本质上是一条“测位并更新标志”的指令。 

1. 先记住一句话 
   

 BIT 做了两件事： 

  - 用 A & M 的结果来设置 Z 
  - 直接把内存里的 bit7、bit6 复制到 N、V 
    

 但它： 

  - 不修改 A 
  - 不修改内存 
  - 不修改 C 
    

 这里 M 指操作数地址里的那个字节。 

---------

2. 它到底在干什么 
   

 先看 Z： 

 Z = ((A & M) == 0) 

 意思是： 

  - 如果 A 和内存值在二进制上“没有任何重叠的 1 位”，那么 Z = 1 
  - 只要有任意一位同时为 1，Z = 0 
    

 所以 BIT 有点像“只做测试、不保留 AND 结果的 AND”。 

 再看 N 和 V： 

  - N = (M & 0x80) != 0 
  - V = (M & 0x40) != 0 
    

 注意这里不是看 A，而是直接看内存值本身。 

---------

3. 一个例子 
   

 假设： 

 A = 0b0000_1111 
 M = 0b1100_1000 

 先算： 

 A & M = 0b0000_1000 

 不是 0，所以： 

  - Z = 0 
    

 再看 M 自己： 

  - bit7 = 1，所以 N = 1 
  - bit6 = 1，所以 V = 1 
    

 最终结果： 

  - A 不变 
  - Z = 0 
  - N = 1 
  - V = 1 
    
---------

4. 再看一个更容易错的例子 
   

 A = 0b0000_0001 
 M = 0b1100_0000 

 则： 

 A & M = 0 

 所以： 

  - Z = 1 
    

 但同时： 

  - N = 1 
  - V = 1 
    

 这说明一件很重要的事： 

  - Z 看的是 A & M 
  - N/V 看的是 M 本身 
    

 这三者不是同一个来源。 

---------

5. 为什么会有这种设计 
   

 因为很多时候程序不是想“算一个值”，而是想“检查某些位”。

 比如一个寄存器里： 

  - bit7 表示状态 A 
  - bit6 表示状态 B 
  - 其他位是别的信息 
    

 这时 BIT 非常方便： 

  - N/V 可以直接反映高两位 
  - Z 可以用来判断 A 关心的那些位有没有命中 
    

 所以它本质上是一条“状态探测指令”。 

---------

6. 在 NES 里为什么它重要 
   

 后面你会接触 PPU 寄存器，尤其是状态寄存器。 
 虽然具体细节以后再展开，但你现在先建立一个意识： 

  - 某些寄存器的高位本来就是“状态位” 
  - BIT 很适合检测这种寄存器内容 
    

 所以它不是冷门指令，而是很实用。 

---------

7. 6502 里的寻址方式 

 标准 BIT 你现在先记两种： 

  - BIT zp 
  - BIT abs 
    

 也就是： 

 BIT \$20 
 BIT \$2002 

 没有你前面常见的 IMM 版本。 
 所以你现在实现时也很简单，只要支持： 

  - ZP0 
  - ABS 
    

 就够了。 

---------

8. 你在 Rust 里该怎么实现 
   

 这条指令其实很适合写成一个小函数： 

```rust
 fn bit(&mut self, addr: u16, bus: &mut impl CPUBus) 
```

 逻辑就是： 

```rust
 let data = bus.cpu_read(addr); 
 let result = self.a & data; 

 self.p.z = result == 0; 
 self.p.v = (data & 0x40) != 0; 
 self.p.n = (data & 0x80) != 0; 
```

 注意： 

  - 不要改 self.a 
  - 不要改 self.p.c 
  - 不要写回内存 
    
---------

9. 在 exe_inst 里怎么接 
   

 和 CMP 这类读内存指令很像： 

```rust
OpCode::BIT => { 
    let operand = self.require_operand(inst.1, bus); 
    self.bit(operand.addr, bus); 
} 
```

 因为 BIT 没有翻页加周期问题，这里不需要额外周期逻辑。 

---------

10. opcode 表里你应该关心哪两个 
    

 6502 正规 BIT： 

  - 0x24 -> BIT ZP0 -> 3 cycles 
  - 0x2C -> BIT ABS -> 4 cycles 
    

 你可以顺手检查你现在的表里是不是已经填了这两个；如果 opcode 已经有了但 exe_inst 还没分发，那就把执行逻辑补上。 

---------

  11. 这条指令最容易写错的点 
      

把 BIT 写成 A &= M 
这是错的。A 完全不该变。 

把 N/V 从 A & M 的结果里取 
这也错。N/V 来自 M 本身的 bit7/bit6。 

不小心改了 C 
BIT 不碰 C。 

---------

12. 你现在至少该写这几个测试 
    

BIT zp：A & M != 0 时，Z = 0 

BIT zp：A & M == 0 时，Z = 1 

BIT abs：当 M 的 bit7/bit6 为 1 时，N/V 应被置 1 

证明 BIT 不会修改 A 

证明 BIT 不会修改 C 

 例如最后一个测试特别有价值： 

  - 先把 C = true 
  - 执行 BIT 
  - 检查 C 仍然为 true 
    
---------

13. 这节课和你前面的知识怎么连起来 
    

 你前面已经学了： 

  - AND：真的做位与，并把结果写回 A 
  - CMP：做比较，只更新标志 
  - ASL/LSR/ROL/ROR：操作位，并更新标志 
    

 那 BIT 可以看成： 

  - “像 AND 一样读取位关系” 
  - “像 CMP 一样不保留结果，只更新标志” 
    

 所以它其实是你现有知识的组合，不是新世界。
