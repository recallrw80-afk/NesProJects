第31课：Sprite Attribute 字节到底在编码什么 

这节课我们只讲一个字节，但它非常关键。 
因为它说明了一件事： 

- 同一个 tile 图，不只是“画出来” 
- 它还能被“怎么画”，而这些规则就塞在属性字节里 


先给核心结论： 


- Sprite 的属性字节不是颜色本身
- 它是一个“渲染控制包” 
- 里面通常包含： 
- 调色板选择 
- 优先级 
- 水平翻转 
- 垂直翻转 


---------

1. 先把属性字节当成一个 8 位开关包 


你可以先把它想成： 

bit7 bit6 bit5 bit4 bit3 bit2 bit1 bit0 

不是每一位都给你“颜色”，而是不同位承担不同控制职责。 

当前阶段你先重点记 4 类信息： 


1. 调色板编号 
2. 优先级 
3. 水平翻转 
4. 垂直翻转 


---------

2. 调色板选择 


属性字节中的低位部分通常用于表示： 


- 这个 sprite 该使用哪一组 sprite palette 


注意，是 sprite palette 组编号，不是最终颜色值。 
这和你前面学 Attribute Table 很像： 


- 它们都不是直接给 RGB 
- 它们都是“选择哪组调色板” 


所以你现在应该开始形成一个统一认识： 


- NES 里很多“颜色信息”其实不是直接颜色，而是“palette 选择编号” 


---------

3. 优先级 


属性字节里还有一位用于表示： 


- 这个 sprite 应该在背景前面还是后面显示 


你现在先不用死抠具体位值，只要理解这个意思： 


- sprite 不一定永远盖在背景上 
- 有些时候可以让它看起来像“躲在背景后面” 


所以属性字节不只是“图形外观”，还控制图层关系。 


---------

4. 水平翻转 


这很直观： 


- 原本 tile 从左往右显示 
- 开启后左右镜像 


这非常有用，因为： 


- 同一张角色朝右的图 
- 只靠水平翻转 
- 就能得到朝左版本 


这样就不需要再存一份完整的新图。 

这也是 NES 节省图形资源的一个典型例子。 


---------

5. 垂直翻转 


和水平翻转类似： 


- 原本从上往下显示 
- 开启后上下镜像 


虽然实际游戏里不一定像水平翻转那么常用，但它同样是“复用图块”的工具。 

所以你可以把翻转理解成： 


- 不用改 Pattern Table 原始图，也能改变显示结果 


---------

6. 为什么属性字节很重要 


因为 OAM 如果只有： 


- 坐标 
- tile 编号 


那精灵就太死板了。 

有了属性字节，同一个 sprite tile 就能： 


- 换 palette 
- 改前后层级 
- 左右翻 
- 上下翻 


所以属性字节让 sprite 系统从“会显示图块”，升级成“能灵活地显示对象”。 


---------

7. 你可以把它和背景系统做个类比 


背景那边你已经学过： 


- tile 编号不等于最终像素
- 还要结合 Attribute Table 和 Palette 


精灵这边也类似： 


- tile 编号不等于最终显示结果 
- 还要结合属性字节和 sprite palette 


所以你会发现 NES 的设计风格很统一： 


- 原始图块数据 
- 再加描述信息 
- 最后组合成显示效果 


它很少把“最终结果”直接存死。 


---------

8. 当前阶段你最该记住的不是具体位值，而是职责分组 


你现在先不用死背 bit5、bit6、bit7 到底是谁。 
先记住属性字节管的是这 4 类事： 


- palette 选择 
- 优先级 
- 水平翻转 
- 垂直翻转 


等你真正进入 sprite 像素生成时，再去绑定具体位编号最合适。 


---------

9. 为什么这节课会帮助你后面理解渲染 


因为后面你做 sprite 渲染时，光从 OAM 里拿到 tile 编号还不够。 
你还必须根据属性字节决定： 


- 这个 sprite 取哪组 palette 
- 图块坐标是否反向采样 
- 最终像素和背景谁在前谁在后 


也就是说，属性字节会直接参与“最终像素怎么出”。 


---------

10. 这节课的最简总结 


- Sprite 属性字节不是颜色本身 
- 它是一个渲染控制包 
- 至少包含： 
- palette 选择 
- 优先级 
- 水平翻转 
- 垂直翻转 
- 它让同一个 tile 图可以更灵活复用和显示
