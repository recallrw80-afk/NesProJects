第 2 课
 
  这节课补 6 个后面一定会反复出现的基础词：      
 
  - 寄存器
  - 地址空间   
  - 内存映射   
  - 时钟  
  - 中断  
  - DMA   
     

  你把这 6 个词吃透，后面学 CPU、PPU 就不会总像在听黑话。 
 
---------

  1. 寄存器是什么            
     

  先用程序员能接受的说法：   
 
  寄存器 = 芯片内部的一小块、超快、用途明确的状态存储
 
  你可以把它理解成“CPU/PPU 自己随身带的几个小变量”。 
 
  比如 CPU 里的这些寄存器，后面会详细学：        
 

  - A：累加器  
  - X、Y：索引寄存器         
  - PC：程序计数器           
  - SP：栈指针 
  - P：状态寄存器            
     

  它们和普通内存的区别：     
 
  - 普通内存大，但访问慢     
  - 寄存器很少，但访问非常快，而且语义固定       
     

  你可以先把 CPU 想成：      
 
  struct Cpu { 
      a: u8,   
      x: u8,   
      y: u8,   
      sp: u8,  
      pc: u16, 
      p: Status,             
  }  
 
  这就是为什么我一直说，模拟器本质上是在维护“硬件状态”。  
 

---------

  2. 地址空间是什么
     

  你平时写程序时，容易把“内存”想成一整块连续数组。   
  NES 里不能这么想。         
 
  地址空间 = CPU 眼中“可以访问的地址范围”        
 
  对 NES CPU 来说，它能发出的地址是 16 位，所以范围是：   
 

  - 0x0000 到 0xFFFF         
     

  这叫 64KB 的 CPU 地址空间。
 
  注意，这句话非常重要：     
 
  地址空间不等于物理上真的有 64KB RAM            
 
  地址空间只是“CPU 能报出的门牌号范围”。         
  这些门牌号后面可能对应：   
 
  - 内部 RAM   
  - PPU 寄存器 
  - 手柄寄存器 
  - APU 寄存器 
  - 卡带 ROM   
  - 卡带 RAM   
  - 甚至是空洞区域           
     

  所以“地址空间”本质上是一张地图，不是一整块纯 RAM。 
 
---------

  3. 内存映射是什么          
     

  有了地址空间，就有“这些地址到底归谁”。         
 
  这就是内存映射。           
 
  在这个项目里，core/bus.c:31 已经把 CPU 地址空间分段了： 
 
  - 0x0000..0x1FFF：RAM      
  - 0x2000..0x3FFF：PPU      
  - 0x4014：DMA
  - 0x4016..0x4017：手柄     
  - 更高地址很多交给卡带     
     

  这就是：
 
  同一个 read(addr)，根据 addr 不同，去不同硬件那里取数据 
 
  也就是 core/bus.c:55 这类逻辑的意义：          
 

  - 地址落在 RAM，就读 RAM   
  - 地址落在 PPU，就转给 PPU 
  - 地址落在 JOY，就转给手柄 
  - 地址落在卡带，就转给 Cartridge 
     

  所以“内存映射 I/O”这个词你可以理解为：         
 
  把硬件寄存器也放进地址空间里，让 CPU 像读内存一样读硬件 
 
  这就是为什么 CPU 读 0x2002，本质上不是在读某个普通数组，而是在读 PPU 状态寄存器。   
 

---------

  4. 镜像是什么
     

  这是地址空间里的第一个容易卡住的点。           
 
  NES 的一部分地址范围很大，但真实硬件并没那么大，于是会重复映射。           
 
  比如这个项目里 RAM 实际只有 0x800 = 2KB，但 CPU 地址里给了 0x0000..0x1FFF 这一大段。
  所以代码里用的是：         
 
  core/bus.c:59
 
  __ram[addr & 0x7FF]        
 
  意思是：
 

  - 0x0000
  - 0x0800
  - 0x1000
  - 0x1800
     

  这些地址最后会落到同一块 2KB RAM 上。          
 
  这就叫镜像。 
 
  你可以把它想成：           
 
  “一栋楼只住了 2 层人，但门牌号印了 8 层，后面几层其实重复指回前面。”       
 

---------

  5. 时钟是什么
     

  你不用先把它想成电子信号。 
  先把它理解成：             
 
  时钟 = 硬件状态推进的节拍器
 
  真实硬件不是“想到哪算到哪”，而是一拍一拍往前走。   
 
  在模拟器里，所谓“推进时钟”，其实就是：         
 

  - 执行一个 CPU 周期        
  - 执行一个 PPU 周期        
  - 执行一个 APU 周期        
  - 更新内部状态             
     

  你可以先把它粗略理解成：   
 
  cpu.clock(); 
  ppu.clock(); 
  ppu.clock(); 
  ppu.clock(); 
 
  这个项目在一帧循环里就是这么干的，core/libretro.c:41 很典型：
 

  - 如果 DMA 正在进行，就先跑 DMA  
  - 否则跑一次 CPU           
  - 然后跑 3 次 PPU          
     

  你现在先记住这个近似原则： 
 
  NES 里常见的节奏是 CPU 1 步，PPU 3 步          
 
  后面讲精确时序时会更细，但现在先建立节拍器概念。   
 

---------

  6. 中断是什么
     

  中断 = 硬件主动打断 CPU，说“先处理我这个事”    
 
  正常情况下 CPU 是顺序执行程序的。
  但有些硬件事件很重要，不能等它慢慢跑到某行代码再说，于是就会触发中断。     
 
  在 NES 里你后面最常见的是：
 

  - NMI   
  - IRQ
     

  先只讲作用，不讲细节。     
 
  NMI

  - 常和 PPU 有关            
  - 比如一帧画面某个关键时刻到了，通知 CPU：“现在可以来更新画面数据了”       
     

  IRQ
 
  - 可能来自 APU，也可能来自某些 Mapper          
  - 用来通知 CPU 某个更一般的事件  
     

  在这个项目里，主循环里就已经体现了这件事：     
 
  - ppu_nmi_triggered() 后调用 cpu_nmi()：core/libretro.c:52   
  - cart_irq() 后调用 cpu_irq()：core/libretro.c:58
     

  所以你现在可以先把中断理解成：   
 
  不是 CPU 自己想起来去做，而是别的硬件发通知让 CPU 立刻切过去处理           
 

---------

  7. DMA 是什么
     

  DMA = Direct Memory Access，直接内存访问       

  先说人话：   

  本来如果 CPU 想搬 256 字节数据，它得自己一字节一字节读再写，很慢。         
  DMA 的意思是：             

  “这次搬运别让 CPU 一步一步亲自搬了，交给专门机制快速搬。”    

  NES 里最经典的是 OAM DMA。 
  用途是把一整页精灵数据快速拷到 PPU 的精灵内存。

  你先不用记 OAM 是什么，只要知道：

  - 精灵数据很多             
  - 一帧里要及时搬过去       
  - 让 CPU 手搬太慢          
  - 所以有 DMA 
     

  这个项目里对应的是写 0x4014 触发 DMA：core/bus.c:91

  case OAM_DMA: dma_active(data, 0); break;      

  意思是：
  CPU 往这个地址一写，就不是普通写寄存器了，而是启动一次 DMA 操作。          

  所以 DMA 的本质你可以记成：

  为了高效搬数据，临时接管一段总线/时序的硬件机制

---------

  8. 把这 6 个概念串起来     
     

  现在把它们连成一句完整的话：
 
  CPU 内部有一些寄存器；     
  CPU 在自己的地址空间里发出读写地址；           
  总线按照内存映射规则，把这些地址转给 RAM、PPU、手柄、卡带等设备；          
  整个系统在时钟驱动下一步一步推进；             
  在关键时刻，PPU 或 Mapper 可以通过中断通知 CPU；   
  当需要大块搬运数据时，可以通过 DMA 加速。      
 
  如果这段话你能顺着复述出来，第 2 课就算真的懂了。

  9. 用一个极小例子理解      
     

  假设 CPU 正在执行代码，突然想读 0x4016。       

  会发生什么？ 

  1. CPU 发出“我要读地址 0x4016”   
  2. Bus 看地址，发现这是手柄寄存器
  3. Bus 不去读 RAM，而是去读控制器状态          
  4. 返回当前这一位按键数据给 CPU  
     

  这就是为什么：             

  CPU 看起来像在读内存，实际上是在和外设通信     

  再看另一个例子：CPU 往 0x4014 写值。           

  1. CPU 发出“我要写地址 0x4014，值是 xx”        
  2. Bus 看地址，发现这是 DMA 寄存器             
  3. 这不是普通变量赋值      
  4. 系统启动一次 DMA，把一页数据搬到 PPU 精灵内存   
     

  这就是“写地址”背后可能触发复杂硬件行为。       
 
---------

  10. 你现在最容易混淆的点   
          

  我提前帮你排雷。           

  误区 1：地址空间 = RAM 大小
  不是。地址空间是 CPU 能报出的地址范围，不代表每个地址都接真实 RAM。        

  误区 2：寄存器 = 普通变量  
  在模拟器实现里它们会表现为普通字段，但语义上它们是硬件状态，不是随便定义的业务变量。

  误区 3：中断 = 函数回调    
  它有点像，但本质上更接近“硬件强制改变 CPU 执行流程”。   

  误区 4：DMA = memcpy       
  结果上像批量拷贝，但它是带时序意义的硬件行为，不只是一个普通库函数。       

---------

这节课你必须记住的 5 句话    

- 寄存器是芯片内部的小而快的状态存储。        

- 地址空间是 CPU 眼里的门牌号范围，不等于真实 RAM。    

- 内存映射决定“这个地址到底归谁响应”。        

-  时钟是硬件状态推进的节拍器。  

- 中断和 DMA 都是在“正常 CPU 顺序执行”之外插进来的硬件机制。
