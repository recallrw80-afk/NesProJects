第 12 课：CMP 和分支，CPU 怎么开始“做判断”
 你现在的 CPU 已经能： 

  - 读写数据 
  - 做位运算 
  - 做寄存器传递 
  - 跳转 
  - 用栈保存返回地址 

 但还缺一个关键能力： 

 根据条件改变执行路径 

 这就是： 

  - 比较指令：CMP、CPX、CPY 
  - 分支指令：BEQ、BNE、BCC、BCS、BMI、BPL、BVC、BVS 

 这组一补上，程序就真的开始像“程序”了。

---------

1. 比较指令本质上不保存结果，只改 flag 

 先看最重要的一句： 
 CMP 不会把结果写回 A，它只是做一次减法式比较，然后更新 flag。 
 也就是： 
 A ? M 
 它的效果更接近： 
 let result = a.wrapping_sub(m); 
 然后根据这个结果更新状态位，但 A 本身不变。 
 同理： 

  - CPX 比较 X 
  - CPY 比较 Y 

---------

2. CMP 要更新哪些 flag 

 最关键的是这三个： 
  - C 
  - Z 
  - N 

 规则先记成下面这样： 
 Z 

  - 如果相等，Z = 1 

 C 
  - 如果左边大于等于右边，C = 1 
  - 这里可以理解成“减法没有借位” 

 N 
  - 看减法结果的 bit7 

 也就是： 

```rust
fn cmp_like(&mut self, left: u8, right: u8) { 
    let result = left.wrapping_sub(right); 
    self.p.c = left >= right; 
    self.p.z = left == right; 
    self.p.n = (result & 0x80) != 0; 
}   
```

 然后：

  - CMP 用 self.a 
  - CPX 用 self.x 
  - CPY 用 self.y 

---------

3. 为什么 CMP 这么重要 
   

 因为分支指令不自己“计算条件”，它们只是读 flag。 

 比如： 

 CMP #$10 
 BEQ equal_case 

 这套逻辑的意思不是： 

  - BEQ 自己再去比较一次 
    

 而是： 

  - CMP 先把结果写进 Z/C/N 
  - BEQ 只看 Z 
    

 所以你要养成这个思维： 

 比较指令负责留下判断结果，分支指令负责读取判断结果。 

---------

4. BEQ / BNE 是最值得先做的分支 
   

 先从这两个开始最合适： 

  - BEQ：Z == 1 就跳 
  - BNE：Z == 0 就跳 
    

 它们最常见，也最容易验证。 

 你现在已经有了 REL 寻址：

```rust
fn rel(&mut self, bus: &mut impl CPUBus) -> u16 { 
    let offset = self.fetch_byte(bus) as i8; 
    self.pc.wrapping_add_signed(offset as i16) 
} 
```

 所以你已经能算出“分支目标地址”。 

 那 BEQ/BNE 的核心就很简单： 

```rust
fn beq(&mut self, target: u16) { 
    if self.p.z { 
        self.pc = target; 
    } 
} 

fn bne(&mut self, target: u16) { 
    if !self.p.z { 
        self.pc = target; 
    } 
} 
```



---------

5. 分支为什么和 JMP 不一样 
   

 JMP 是无条件跳。 

 BEQ/BNE 是条件跳。 

 区别就在于： 

  - JMP 一定改 PC 
  - BEQ/BNE 只有条件满足才改 PC 
    

 但两者有个共同点： 

 它们真正做的事都是改 PC 

 所以你现在可以把“流程控制类指令”统一理解成： 

 决定 PC 是否改到别处 

---------

6. 实现顺序建议 
   

 你现在最稳的顺序是： 

  1. 先补 cmp_like(left, right) 
  2. 实现 CMP 
  3. 实现 CPX 
  4. 实现 CPY 
  5. 实现 BEQ 
  6. 实现 BNE 
     

 这就够你开始跑最小条件逻辑了。 

 如果这组稳了，再加： 

  - BCC/BCS 读 C 
  - BMI/BPL 读 N 
  - BVC/BVS 读 V

7. 你现在代码里怎么接 
   

 你已经有： 

  - require_operand_addr(inst.1, bus) 
  - resolve_operand_addr(REL, ...) 
  - set_zn() 
  - self.p.c / z / n / v 
    

 所以加这组不会很痛。 

 你只需要在 exe_inst() 里加： 

 OpCode::CMP => { 
 let addr = self.require_operand_addr(inst.1, bus); 
 self.cmp(addr, bus); 
 } 
 OpCode::CPX => { 
 let addr = self.require_operand_addr(inst.1, bus); 
 self.cpx(addr, bus); 
 } 
 OpCode::CPY => { 
 let addr = self.require_operand_addr(inst.1, bus); 
 self.cpy(addr, bus); 
 } 
 OpCode::BEQ => { 
 let addr = self.require_operand_addr(inst.1, bus); 
 self.beq(addr); 
 } 
 OpCode::BNE => { 
 let addr = self.require_operand_addr(inst.1, bus); 
 self.bne(addr); 
 } 

 然后实现： 

 fn cmp_core(&mut self, left: u8, right: u8) { 
 let result = left.wrapping_sub(right); 
 self.p.c = left >= right; 
 self.p.z = left == right; 
 self.p.n = (result & 0x80) != 0; 
 } 

 fn cmp(&mut self, addr: u16, bus: &mut impl CPUBus) { 
 let value = bus.cpu_read(addr); 
 self.cmp_core(self.a, value); 
 }

 fn cpx(&mut self, addr: u16, bus: &mut impl CPUBus) { 
 let value = bus.cpu_read(addr); 
 self.cmp_core(self.x, value); 
 } 

 fn cpy(&mut self, addr: u16, bus: &mut impl CPUBus) { 
 let value = bus.cpu_read(addr); 
 self.cmp_core(self.y, value);
 } 

 fn beq(&mut self, target: u16) { 
 if self.p.z { 
 self.pc = target; 
 } 
 } 

 fn bne(&mut self, target: u16) { 
 if !self.p.z { 
 self.pc = target; 
 } 
 }

8. 先写哪些测试 
   

 最少先补这 5 个： 

  1. CMP #imm 相等时： 
     
  - Z = 1 
  - C = 1 
  - N = 0 
    
  2. CMP #imm 小于时： 
     
  - Z = 0 
  - C = 0 
    
  3. CMP #imm 大于时： 
     
  - C = 1 
  - Z = 0 
    
  4. BEQ 在 Z = 1 时跳转 
  5. BNE 在 Z = 0 时跳转 
     

 如果你愿意再多补两个： 

  6. BEQ 在 Z = 0 时不跳 
  7. BNE 在 Z = 1 时不跳 
     
---------

9. 这一课你最需要记住的 4 句话 
   

CMP/CPX/CPY 不写回寄存器，只更新 flag。 

比较的本质是一次减法式判断。 

分支指令不负责计算条件，它们只读取 flag。 

分支和跳转的本质都是“是否修改 PC”。
