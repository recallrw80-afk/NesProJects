第15课：ASL / LSR / ROL / ROR

 这节课的目标很明确：把 6502 里的“位操作”真正看懂，并且知道它们为什么和 Carry 强相关。学完这节，你就能自己把这 4 条指令接进当前 CPU 框架。 
1. 先建立直觉 

 你可以先把一个 u8 想成 8 个格子： 

 bit7 bit6 bit5 bit4 bit3 bit2 bit1 bit0 

 所谓“移位”，就是把这些位整体往左或往右推一格。 
 问题在于：被挤出去的那一位去哪了？新空出来的位置又填什么？ 

 6502 的答案就是： 

  - 被挤出去的位，进 Carry 
  - 空出来的位置，要么补 0，要么补“旧的 Carry” 

 这就分成了两类： 
  - ASL / LSR：普通移位，补 0 
  - ROL / ROR：带 Carry 的旋转 

---------

2. ASL：Arithmetic Shift Left 

 语义：
  - 原来的 bit7 进入 C 
  - 结果整体左移 1 位 
  - 新的 bit0 补 0 

 例如： 
 原值: 0100_0011 
 ASL: 1000_0110 
 C = 0 
 再看一个： 
 原值: 1100_0011 
 ASL: 1000_0110 
 C = 1 
 你先别管名字里为什么叫 “Arithmetic”，在 NES 里你可以先把它理解成“左移”。
 ASL 更新的标志位： 

  - C：原 bit7 
  - Z：结果是否为 0 
  - N：结果 bit7 是否为 1 

---------

3. LSR：Logical Shift Right 

 语义： 
  - 原来的 bit0 进入 C 
  - 结果整体右移 1 位 
  - 新的 bit7 补 0 

 例如： 
 原值: 1000_0011 
 LSR: 0100_0001 
 C = 1 
 LSR 更新的标志位： 

  - C：原 bit0 
  - Z：结果是否为 0 
  - N：结果 bit7 是否为 1 

 这里有个很重要的结论： 
  - LSR 执行后，N 一定是 0 

 因为最高位永远补 0。 

---------

4. ROL：Rotate Left 

 这是第一次把 Carry 真正当成“额外 1 位”看。 
 语义： 

  - 原来的 bit7 进入 C 
  - 结果左移 1 位 
  - 新的 bit0 填入“旧的 Carry” 

 例如，假设旧 C = 1： 

 原值: 0100_0011 
 ROL : 1000_0111 
 C = 0 

 因为： 

  - 左移后本来是 1000_0110 
  - 但最低位不是补 0，而是补旧 C=1 
  - 所以结果变成 1000_0111 

 更新标志位： 
  - C：原 bit7 
  - Z：结果是否为 0 
  - N：结果 bit7 是否为 1 

 关键点只有一个： 
  - ROL 用的是“操作前的 Carry”，不是刚算出来的新 Carry 

 这个细节很容易写错。 

---------

5. ROR：Rotate Right 

 和 ROL 对称。 
 语义： 

  - 原来的 bit0 进入 C 
  - 结果右移 1 位 
  - 新的 bit7 填入“旧的 Carry” 

 例如，假设旧 C = 1： 
 原值: 0100_0011 
 ROR : 1010_0001
 C = 1 
 因为： 

  - 右移后本来是 0010_0001 
  - 最高位补旧 C=1 
  - 所以结果变成 1010_0001 

 更新标志位仍然是： 
  - C 
  - Z 
  - N 

---------

6. 这 4 条指令和 ADC/SBC 的区别 

 你前面学 ADC/SBC 时，Carry 更像“进位/借位标志”。 
 但在 ROL/ROR 里，Carry 更像“第9位寄存器”。 
 所以你可以这样记： 

  - ADC/SBC：Carry 参与算术 
  - ROL/ROR：Carry 参与位旋转 

---------

7. 这 4 条指令有两种操作对象 

 6502 里它们既可以操作累加器，也可以操作内存。 
 例如： 
 ASL A 
 LSR A 
 ROL A 
 ROR A 
 这是改 A 寄存器。 
 而： 
 ASL \$20 
 ROR \$1234 
 这是改某个内存地址里的值。 
 所以你实现时，最好不要把逻辑写死在 A 上，而是拆成两层： 

  1. “给一个 u8，算出结果并更新 flags” 
  2. “这个 u8 来自 A，还是来自内存” 

 这是和你现在架构最匹配的做法。 

---------

8. 推荐实现方式 

 你可以先写 4 个小 helper，专门处理一个字节： 
```rust
 fn op_asl(&mut self, value: u8) -> u8
 fn op_lsr(&mut self, value: u8) -> u8 
 fn op_rol(&mut self, value: u8) -> u8 
 fn op_ror(&mut self, value: u8) -> u8 
```

它们负责： 

  - 计算新值 
  - 设置 C 
  - 设置 Z/N 
  - 返回结果 

 然后在执行指令时分两种情况： 
  - ACC：直接改 self.a 
  - 其他寻址：读内存 -> helper -> 写回内存 

 伪代码就是： 

```rust
match mode { 
    AddrMode::ACC => { 
        self.a = self.op_asl(self.a); 
    } 
    _ => { 
        let addr = self.require_operand_addr(bus, mode); 
        let value = bus.read(addr); 
        let result = self.op_asl(value); 
        bus.write(addr, result); 
    } 
}   
```

 另外 3 条同理。

---------

9. 最容易写错的 4 个点 
   
     ROL/ROR 必须先保存旧 Carry 
     因为新 Carry 和参与旋转的 Carry 不是同一个时刻的值。 
     
     LSR 后 N 一定是 0 
     但你最好还是统一走 set_zn(result)，这样逻辑更整齐。 
     
     内存型指令一定要写回 
     不是只算结果和改 flags 就完了。 
     
     累加器型和内存型不要混在一起写 
     否则执行分支会越来越乱。 

---------

10. 你现在可以怎么落地 

 建议顺序： 
  1. 先只实现 helper：op_asl/op_lsr/op_rol/op_ror 
  2. 先接 ACC 版本 
  3. 测试过了，再接内存版本 
  4. 最后把对应 opcode 填完整 

 这样最稳。 

---------

11. 这一课的最小测试集 

 你至少要写这些测试： 
  - ASL A：0b1000_0001 -> 0b0000_0010，C=1 
  - LSR A：0b0000_0001 -> 0b0000_0000，C=1，Z=1 
  - ROL A：旧 C=1，0b0100_0000 -> 0b1000_0001 
  - ROR A：旧 C=1，0b0000_0010 -> 0b1000_0001 
  - 一个内存版测试：例如 ASL $20，确认总线上的值被改写 
  - 一个边界测试：LSR 后 N=0 

---------

12. 你先记住一句话 

  - ASL/LSR：移出去的位进 Carry，空位补 0 
  - ROL/ROR：移出去的位进 Carry，空位补“旧 Carry” 

 这就是这节课的核心。 
 课后问题 

  1. 为什么 ROL 和 ASL 的区别不只是“名字不同”？ 
  2. 为什么实现 ROR 时必须先保存旧 Carry？ 
  3. LSR 之后，N 为什么通常一定为 0？ 
  4. 如果执行 ASL A 前 A = 0x80，执行后 A、C、Z、N 分别是什么？ 

 你先回答这 4 个问题。答完后，如果你准备直接写代码，我下一条就按你现在的 Rust 结构，带你把这 4 条指令接进 cpu/mod.rs。 

---

1. ASL是算术移位，ROL是环绕移位，前者只需补0，后者需要将旧的C标志作为补位。
2. ROR需要将旧的C标志写入移位后的补位，被移出那一位会作为新值写入C标志，所以必须先保存旧值。
3. LSR是右移移位，结果的bit7 必为0，因此N也必为0。
4. 如果ASL A前A = 0x80，也就是bit7 = 1，它写入新的C = 1，原值左移后的值的0，因此A = 0，Z = 1，N = 0。
